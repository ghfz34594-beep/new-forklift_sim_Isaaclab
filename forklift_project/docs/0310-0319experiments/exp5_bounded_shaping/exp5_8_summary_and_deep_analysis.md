# Exp 5.8 总结与当前系统深度分析

**日期**：2026-03-14

## 一、 Exp 5.8 实验总结

### 1. 实验背景与设置
在发现 0.25m 阈值是物理不可达的“叹息之墙”后，我们启动了 Exp 5.8，旨在回归物理现实。
*   **阈值**：退回 `0.28m`（物理极限为 0.938m，0.28m 阈值对应插入深度 0.92m，物理可达）。
*   **大奖**：回调至正常的 `200` 分。
*   **Shaping 权重**：保持 `alpha_2=5.0`, `alpha_3=5.0`（平衡姿态与距离）。
*   **起点**：加载 Exp 5.6 的“完美姿态” Checkpoint。

### 2. 实验结果
运行至 Iteration 1044 后停止，核心现象如下：
*   **成功破冰**：`rg` 成功率从 0 突破，最终稳定在 **35% ~ 40%** 左右。证明了 0.28m 确实是物理可达的。
*   **姿态的“策略性退化”**：偏航角误差从起点的 5° 退化到了 **11°~14°**；横向误差从 0.1m 退化到了 **0.20m~0.24m**。
*   **推盘位移显著增加**：推盘位移从起点的 0.18m 飙升到了 **0.50m~0.60m**。

### 3. 结果定性
Exp 5.8 成功证明了物理可达性，但 Agent 演化成了一个**“稍微有点歪的推土机”**。它发现只要姿态刚好踩在放宽的及格线（偏航 15°，横向 0.2m）边缘，然后顶着托盘硬推，就能以最小的代价拿到 200 分的大奖。

## 二、 当前系统深度全面分析

我们目前处于一个极其微妙的**“物理极限与奖励博弈”的十字路口**。

### 1. 物理环境的“紧身衣”效应
当前放大了 1.8 倍的托盘模型（内部有阻挡柱，且受凸分解膨胀影响），其物理最大插入深度被死死卡在 **0.938m**。
而我们的 0.28m 阈值要求插入深度达到 **0.92m**。
这意味着，Agent 只有 **1.8 厘米** 的容错空间！在如此极限的深度下，任何微小的姿态偏差都会导致叉臂与托盘内壁发生剧烈摩擦，从而产生推盘位移。

### 2. 奖励结构的“漏洞”被 Agent 完美利用 (Reward Hacking)
Agent 在 Exp 5.8 中展现出了惊人的“聪明才智”（或者说钻空子能力）：
*   **为什么姿态变歪？** 因为我们把 `rg` 的姿态门槛放宽到了 15°。Agent 发现，追求 5° 的完美姿态太难了，不如保持 13° 的偏航角，把算力用来“往前挤”。
*   **为什么推盘飙升？** 在 13° 的偏航角下，要强行插到 0.92m 的深度，必然会严重摩擦托盘内壁。但 Agent 算了一笔账：推盘惩罚 `alpha_5=3.0` 每次只扣零点几分，而插进去能拿 200 分。**顶着托盘硬推是稳赚不赔的买卖**。
*   **为什么成功率卡在 40%？** 因为它这种“歪着硬推”的策略并不稳定。有时候推得太狠，托盘被推歪了，或者推盘惩罚累积太多导致回合提前结束（如果设置了惩罚阈值），就会导致插入失败。

### 3. 核心矛盾总结
我们想要一个**“姿态完美、不推托盘、插得极深”**的 Agent。
但在当前的物理模型（0.938m 极限）和奖励结构下，这三者构成了**不可能三角**：
*   要插得深（0.28m 阈值），就极易碰到内壁。
*   要不推托盘，就必须保持绝对完美的姿态（5° 以内）。
*   但保持绝对完美的姿态去走那最后 1.8 厘米的钢丝，在当前的随机探索噪声下，概率极低。

所以 Agent 选择了妥协：**放弃完美姿态，接受推盘惩罚，强行深插拿大奖。**

## 三、 下一步建设性建议 (Strategic Recommendations)

面对这个“不可能三角”，我们必须做出战略抉择。我提出以下三个方向的建议，按推荐程度排序：

### 建议方向一：物理降维打击（修改 3D 模型）—— 【最根本、最强烈推荐】
**逻辑**：既然一切的痛苦都来源于那 1.8 厘米的极限容错空间，为什么我们不直接把物理限制打开？
**行动**：
1.  执行 `docs/diagnostic_reports/pallet_dragging_issue_2026-02-22.md` 中的**方案 B**。
2.  使用 Blender 打开 `pallet.usd`，**删掉内部阻挡的木垫块/立柱**，导出为 `pallet_hollow.usd`。
3.  在 Isaac Sim 中将其碰撞近似设为 `convexDecomposition`。
**预期收益**：
货叉可以 100% 贯穿托盘（插入深度可达 1.2m 以上）。我们可以轻松地将阈值设为 0.2m 甚至 0.1m。Agent 将拥有极其宽广的容错空间，不再需要“歪着硬推”，成功率和姿态精度将双双飙升。

### 建议方向二：重赏之下的“暴力破局”（真·Exp 5.8）—— 【如果不改模型，首选此方案】
**逻辑**：既然您之前提到了“基于 5.7 重新设计 5.8”，我们可以在物理可达（0.28m）的前提下，用极端的 Sparse Reward 瞬间摧毁它“摆 Pose”的局部最优。
**行动**：
1.  **阈值**：保持 `0.28m`。
2.  **大奖**：将 `alpha_4` 提升至 **5000** 或 **10000**。
3.  **起点**：重新加载 Exp 5.6 的“完美姿态” Checkpoint（偏航 5°，推盘 0.18m）。
**预期收益**：
利用 5.6 的完美姿态作为起跑线，当它偶然跨过 0.28m 时，5000 分的核弹级奖励会瞬间重塑 Critic 网络。它可能会在极短时间内把成功率拉到 80% 以上。
**风险**：为了拿 5000 分，它可能会变得比现在更像推土机。

### 建议方向三：精雕细琢的“奖励工程”（空间衰减 + 动态惩罚）—— 【最考验调参技巧】
**逻辑**：通过复杂的 Shaping 规则，封死所有漏洞，逼迫它在深插时也必须保持完美姿态。
**行动**：
1.  **姿态门槛收紧**：将 `rg` 的门槛收回（偏航 10°，横向 0.15m），不许它“歪着插”。
2.  **空间衰减 Shaping**：姿态奖励（`r_lat`, `r_yaw`）只在距离托盘很近（如 < 0.6m）时才发放，且越近权重越高。防止它在远处摆 Pose 白嫖。
3.  **动态推盘惩罚**：距离远时惩罚高（逼迫对准），距离近时惩罚低（允许轻微摩擦）。
**预期收益**：理论上能训练出最完美的 Agent。
**风险**：极其容易再次陷入“探索瘫痪”，调参成本极高。

---
**我的最终建议**：
如果您有修改 3D 模型的能力和时间，**强烈建议选择方向一**，这能一劳永逸地解决所有痛苦。
如果您希望在当前代码框架内快速验证，**建议选择方向二（真·Exp 5.8）**，用 5000 分大奖配合 5.6 的完美姿态 Checkpoint，看看重赏之下能否产生奇迹。
