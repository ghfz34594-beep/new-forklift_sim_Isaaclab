# 256x256 分辨率预训练+微调 (极短冻结期) 实验结论 (2026-03-11)

## 1. 实验背景与目标

在上一轮实验（正常惩罚，冻结 500 代）中，我们发现预训练的视觉特征内生带有 16cm 的横向误差。在长时间冻结 Backbone 的情况下，Actor 无法进行精准对齐，只能收敛到“推着托盘走”的假成功状态。
因此，本轮实验的核心目标是：**大幅缩短冻结期（从 500 代降至 50 代），试图让 RL 的端到端梯度尽早回传，以修正预训练的视觉误差，实现零碰撞的完美插入。**

## 2. 实验配置

- **分支**：`exp/vision_cnn/finetune_256x256`
- **训练日志**：`logs/20260311_154823_train_finetune_256x256_freeze50_env64.log`
- **训练规模**：`num_envs=64`
- **视觉输入**：`256x256`，60 度俯视相机
- **Backbone**：加载 `outputs/vision_pretrain_256x256/best_backbone.pt`
- **冻结策略**：**仅前 50 个 policy iterations 冻结 Backbone**，随后完全解冻。
- **奖励函数**：正常推盘惩罚（`k_pallet_push_pen = 1.0`, `pallet_push_insert_gate = 0.15`）

### 3.2 现象深度剖析 (Catastrophic Forgetting)

本轮实验证明了“用 RL 梯度去修视觉误差”的设想是**失败的**。

1. **冻结期的表现符合预期**：前 49 代，Actor 努力适应带有 16cm 误差的视觉特征。它能把车开到托盘附近（对齐率 50%），也尝试往前靠（推盘位移 0.14m），但因为看不准，始终插不进去（成功率 0%）。
2. **解冻初期的“回光返照”**：在第 50 代解冻时，RL 梯度开始回传，成功率和插入率确实出现了短暂的破零。这说明端到端微调在理论上是能影响对齐精度的。
3. **灾难性遗忘 (Catastrophic Forgetting)**：
   - 在第 50 代时，Actor 根本还没学会怎么开车（成功率只有 1%）。
   - 这时候它传回给 MobileNet 的梯度，不是“如何更精准地看清托盘”，而是充满噪声的“瞎探索”梯度。
   - MobileNet 接收到这些低质量的 RL 梯度后，原本在预训练里学到的几何特征被迅速破坏（洗掉）了。
   - **恶性循环**：眼睛（Backbone）变瞎了 -> 动作（Actor）更插不进去 -> 传回来的梯度更差 -> 眼睛更瞎。
   - 最终，策略退化成了一个只会把车开到托盘附近（对齐率 50%），但既不敢推、也插不进去的“傻子”。

## 4. 结论

**结论：“极短冻结期 + 端到端 RL 微调”的路线走不通。RL 稀疏且高方差的奖励信号，不足以在 Actor 尚未成熟时去指导 Backbone 进行像素级的特征微调，反而会破坏预训练特征。**

我们目前面临一个两难的死局：
- **冻结太久（500代）**：预训练特征有 16cm 误差，Actor 看不准，只能变成“推土机”（假成功）。
- **冻结太短（50代）**：Actor 还没学会开车就解冻，垃圾梯度把 Backbone 洗废了，变成“傻子”（真失败）。

## 5. 破局思路与下一步计划

既然 RL 的梯度靠不住，我们**不能指望 RL 去修视觉误差**。我们必须在**预训练阶段**就把视觉误差降下来！

**下一步计划：回头优化视觉预训练 (Visual Pretraining)**
目标：将预训练的横向误差 (Y) 从目前的 16cm 压低到 **5cm 以内**。只有“眼睛”真的准了，RL 冻结 Backbone 训练才能真正成功。

可行的优化方向：
1. **扩大数据集**：目前的 64k 数据集可能不足以让模型学到极高精度的几何特征，考虑收集 200k 甚至更大的数据集。
2. **升级 Backbone**：MobileNetV3-Small 的容量可能受限，考虑升级到 MobileNetV3-Large 或 ResNet18。
3. **优化预训练损失函数**：针对 Y 和 Yaw 增加更高的权重，或者引入关键点预测等更密集的监督信号。
