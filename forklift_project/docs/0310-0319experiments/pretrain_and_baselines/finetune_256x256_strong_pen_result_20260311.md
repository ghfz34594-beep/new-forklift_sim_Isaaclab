# 256x256 分辨率预训练+微调 (强惩罚) 实验结论 (2026-03-11)

## 1. 实验背景与目标

在证明了“纯 RL 无法直接学习 256x256 图像特征”后，我们启动了本实验。
核心目标是验证：**高分辨率 (256x256) + 预训练特征** 能否解决之前 64x64 分辨率下的“推托盘”陷阱，并提升最终的对齐与插入成功率。

## 2. 实验配置

- **分支**：`exp/vision_cnn/finetune_256x256`
- **训练日志**：`logs/20260311_142441_train_finetune_256x256_env64.log`
- **训练规模**：`num_envs=64`
- **视觉输入**：`256x256`，60 度俯视相机
- **Backbone**：加载 `outputs/vision_pretrain_256x256/best_backbone.pt`
- **冻结策略**：前 500 个 policy iterations 冻结 Backbone
- **奖励函数**：继承了之前为了治“推托盘”而加码的强惩罚配置（`k_pallet_push_pen = 3.0`, `pallet_push_insert_gate = 0.80`）

## 3. 实验过程与结果

该实验运行至 **Learning iteration 194** 时被手动终止。

### 3.1 核心指标变化趋势

实验呈现出非常典型的三个阶段：

| 阶段 | Iteration | `success_rate_total` | `frac_inserted` | `pallet_disp_xy_mean` (推盘位移) | 现象描述 |
|---|---|---|---|---|---|
| **初期爆发** | 0 - 50 | 从 0% 飙升至 **21.3%** | 飙升至 **23.4%** | 高达 1.4m - 1.7m | 预训练特征立刻生效，Agent 迅速学会对齐和插入，但同时也学会了“推托盘”以获取潜力奖励。 |
| **中期受挫** | 50 - 100 | 维持在 20% 左右，后开始下滑 | 维持在 10%-15%，后降至 0% | 从 1.5m 骤降至接近 0m | Critic 学到了强烈的“推托盘惩罚”，开始压制 Actor 的前向动作。 |
| **后期保守** | 100 - 194 | 持续下滑至 **10.4%** | 彻底归零 (**0.0000**) | 彻底归零 (**0.0030m**) | Actor 被高额惩罚“吓破了胆”，为了绝对避免推盘，变得极其保守，连正常的插入动作都不敢做了。 |

### 3.2 现象分析

1. **预训练的巨大威力**：与纯 RL Baseline 跑了 111 代依然是 0 成功率相比，本实验在第 20 代就突破了 20% 成功率。这**无可辩驳地证明了 256x256 预训练特征的有效性**。它让 Agent 瞬间具备了看清托盘并进行对齐的能力。
2. **强惩罚的副作用**：我们之前在 64x64 实验中，因为 Agent “看不清”而加重了推托盘惩罚（权重 3.0，取消浅插入豁免）。现在 Agent “看清”了，但这个过强的惩罚反而成了绊脚石。它导致 Agent 在稍微碰到托盘时就受到严厉打击，最终演变成“宁可不插，也绝不推盘”的极端保守策略。
3. **Backbone 未被破坏**：由于前 500 代 Backbone 是冻结的，后期的性能退化**绝对不是**因为 PPO 梯度破坏了视觉特征，纯粹是控制策略（Actor）在强惩罚下的退让。

## 4. 结论

**结论：256x256 预训练特征完全有效，成功解决了“看不清”的问题；但当前的“强推盘惩罚”配置阻碍了策略的进一步学习。**

## 5. 下一步行动

既然 256x256 已经能提供足够的视觉精度，我们不再需要用极端的惩罚来防止“瞎推”。

**下一步计划**：进行单因素奖励回调实验。
1. 保持 256x256 分辨率和预训练权重。
2. 保持冻结策略（500 iter）。
3. **将推托盘惩罚回调至正常水平**：
   - `k_pallet_push_pen` 从 `3.0` 降回 `1.0`。
   - `pallet_push_insert_gate` 从 `0.80` 降回 `0.15`（恢复浅插入豁免，允许正常的物理接触和微调）。
4. 重新启动 Finetune 训练。
