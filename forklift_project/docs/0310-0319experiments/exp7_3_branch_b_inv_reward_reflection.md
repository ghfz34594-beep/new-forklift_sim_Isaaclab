## 1. 实验结果现象：陷入“动态逃避”

实验运行至 Iteration 1433/2000，Agent 彻底陷入了**“动态逃避（匀速绕圈）”**的局部最优。

### 数据演变过程：
1. **初期 (Iter 50)**：Agent 敢于靠近托盘 (`dist_front` ~ 0.4m)，但姿态不对，频繁撞击托盘，导致推盘位移 (`pallet_disp_xy`) 高达 0.55m，吃到了大量的推盘惩罚 ($r_p$)。
2. **中期 (Iter 300)**：Agent 开始害怕靠近，退到了 0.9m 处，尝试用大偏航角（55°）“斜着蹭”托盘，试图在不触发撞击的情况下拿到距离奖励。
3. **后期 (Iter 1400+)**：Agent 彻底“摆烂”。
   - `dist_front` 退回到了 **1.90m**（几乎是出生点）。
   - `yaw_deg_mean` 达到了 **79.5°**（完全横着开）。
   - `frac_inserted` 降为 **0.0000**（彻底放弃插入）。
   - `rp` 惩罚降至 -0.10（成功规避了撞击）。
   - `ra` 惩罚降至 -0.0003（动作极其平滑，保持匀速）。

### 行为画像：
Agent 变成了一台**“匀速绕圈巡逻车”**。为了不扣停滞分（必须动），为了不扣推盘分（不能靠近），为了不扣动作分（不能急转弯），它选择在距离托盘 1.9m 的安全距离外，横着车身匀速画圈。

## 2. 论文对实验失败的关键启发（核心诊断）

之前我们把失败归咎于“绝对状态奖励不如差分势函数”，这是一种**严重的误判**。经过对论文的重新审视，我们发现了实验绕圈的真正根因：**不是引力不够强，而是引力场太窄，且惩罚项的威慑被不当放大了。**

### 2.1 最被忽视的核心：参考轨迹提供了“三重密集引力”
论文的正奖励包含三个反比例项：
$$R^+ = \alpha_1 \frac{1}{r_d} + \alpha_2 \frac{1}{r_{cd}} + \alpha_3 \frac{1}{r_{c\psi}} + \alpha_4 r_g$$

- **致命问题**：实验中，我们在实现这三项时，都加了防爆常数 $c=0.1$（即 $1/(x+0.1)$），把最大奖励“盖帽”在了 10。
- **后果**：在 1.9m 远处，这三项提供的正奖励总和极低。论文的本意是让 $1/r_{cd}$ 和 $1/r_{c\psi}$ 在全过程（哪怕是远处）都提供密集的正反馈，引导 Agent “沿着轨迹走、把车头摆正”。但因为 $c$ 的存在，远处的引力被极大地削弱了。

### 2.2 惩罚项的精确设计：阈值触发 vs 连续惩罚
论文的四个惩罚项有极其精妙的**阈值设计**，而我们在权重配置上打破了这种平衡：
1. **$r_p$ (推盘惩罚)**：论文是二值惩罚（碰了且速度>0.01就扣 -1，没碰是 0）。我们的代码虽然实现了二值，但给它分配了高达 **3.0** 的权重！在正奖励被 $c=0.1$ 盖帽（远端只有零点几分）的情况下，-3.0 的惩罚是毁灭性的，直接把 Agent “打怕了”。
2. **$r_{ini}$ (停滞惩罚)**：论文有距离门槛（$r_d > 0.3$ 时才惩罚静止）。我们的代码确实实现了这个门槛，但正因为有这个门槛，Agent 发现：只要我在 0.3m 以外（比如 1.9m），我就**必须一直动**，否则就会被扣高达 **20.0** 分的停滞惩罚！
3. **$r_a$ (动作惩罚) vs Boundary Loss**：论文中 $L_{bound} = ||\mu(o_t)||$ 是在损失函数中防止动作饱和的正则化项，**它不会阻止 Agent 做必要的转向**。而我们在奖励中设置了 $r_a$（权重 0.5），导致 Agent 因为“怕扣分”而不敢打方向盘纠偏，最终只能以 79.5° 的偏航角横着开。

### 2.3 Clothoid 曲线：被低估的核心设计
论文特别强调参考轨迹是 **clothoid 曲线（回旋曲线）近似**。
- **特点**：曲率线性变化，从直线平滑过渡到弯道。
- **作用**：对于后轮转向的叉车，Clothoid 配合 $1/r_{cd}$ 和 $1/r_{c\psi}$，相当于铺了一条“从出生点到目标的奖励高速公路”。
- **我们的缺陷**：我们用的是“三次 Bézier + 直线”。这种曲线的曲率变化可能不够平滑，导致在远端无法提供正确的朝向引导（$r_{c\psi}$ 信号混乱）。

### 2.4 综合诊断：绕圈的根因链条
```text
Clothoid引导不足 ──→ 远处正奖励(rcd, rcpsi)方向不明确
        +                      ↓
防爆常数c=0.1 ──→ 远端引力被严重削弱       ↓
        +                      ↓
推盘惩罚过强(3.0) ──→ 靠近就被重罚    → 惩罚 >> 正奖励 → 逃避
        +                      ↓
rini(20.0)逼迫移动 ──→ 必须一直动          → 匀速绕圈
        +
ra压制转向 ──→ 不敢纠偏              → 偏航79.5°横着开
```

## 3. 下一步计划：论文指导下的精准修复

我们**不需要**退回到差分势函数。论文的“Clothoid 轨迹 + 三重反比例引力”本质上就是一种沿轨迹的密集梯度场，功能上等价于差分势函数。我们只需要**修好它**。

| 修复项 | 当前状态 | 论文指导的修改 |
|---|---|---|
| **防爆常数 $c$** | 0.1 | **去掉 $c$**，改用 `torch.clip(reward, 0, MAX_VAL)`，释放 $1/x$ 的原生引力！ |
| **参考轨迹** | Bézier + 直线 | **必须改为 Clothoid 曲线近似**，提供正确的曲率引导。 |
| **$r_p$ 推盘惩罚** | 权重 3.0 | 权重**大幅降低**（如降至 0.5 或 0.1），允许初期试错。 |
| **$r_a$ 动作惩罚** | 权重 0.5 | 权重**降低**（如降至 0.1），释放转向能力。 |
| **$L_{bound}$** | 未实现 | 探索在 RSL-RL 中加入策略均值的 Boundary Loss。 |
| **$1/r_{cd}$ 和 $1/r_{c\psi}$** | 权重 5.0 | 确保其权重不低于终点引力 $1/r_d$。 |

**结论**：这次失败不是因为绝对状态奖励不行，而是我们的实现“画虎画皮难画骨”。下一步，我们将严格按照上述表格，重构 Branch B 的奖励场和轨迹生成。
