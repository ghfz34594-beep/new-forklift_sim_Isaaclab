# 实验 7：消除视觉盲区与稳健泛化训练方案 (2026-03-15)

## 1. 核心问题：视觉盲区导致“推土机”行为
在之前的实验（Exp 5.x, Exp 6.0b）中，我们反复遇到模型在最后插入阶段变成“推土机”的问题。
经过仔细排查，发现根本原因在于**相机俯仰角 (Pitch) 设置不当**。
当前 `camera_rpy_local_deg = (0.0, 60.0, 0.0)`，导致相机的视野（FOV）下边缘无法覆盖货叉根部。当托盘被推到货叉根部时，它会从画面下方**完全消失**。
模型在关键的最后几厘米变成了“瞎子”，只能依靠本体感觉瞎猜，这是导致它无法精确判断“是否插到底”以及产生推盘行为的物理级原因。

## 2. 解决方案：调整相机角度与渲染材质
1. **消除物理盲区**：将相机的俯仰角从 60° 调整为 **75°**。
```python
camera_rpy_local_deg: tuple[float, float, float] = (0.0, 75.0, 0.0)
```
2. **解决渲染降级导致的“隐身”**：在 Headless 训练模式下，由于材质丢失，黑色货叉与深色地面融为一体。我们强制将叉车材质覆盖为**亮红色**，以提供极高的对比度。
```python
visual_material=sim_utils.PreviewSurfaceCfg(
    diffuse_color=(1.0, 0.0, 0.0),  # 强制亮红色
    metallic=0.5,
    roughness=0.5,
)
```

**影响**：这一物理和渲染层面的改动会导致视觉输入的特征分布发生根本性改变。旧的视觉 Backbone 提取的特征将完全失效。因此，**必须从头开始训练（Scratch）**。另外，为了提升特征提取能力，我们将视觉骨干网络从 MobileNetV3-Small 升级为 **ResNet34**。

## 3. 实验 7 训练方案设计 (多阶段课程学习)

鉴于必须从头训练，且我们已经积累了大量关于奖励函数和泛化的经验，Exp 7 将采用**多阶段课程学习 (Curriculum Learning)** 策略，稳扎稳打地培养一个既能精准插入，又具备强泛化能力的模型。

### 阶段一：Exp 7.1 - 核心能力重建（极窄范围 + 核弹奖励）
**目标**：让全新的视觉网络快速学会“在 75° 视角下，把货叉插进托盘”这个核心动作，建立正确的特征映射。
*   **初始范围**：极窄，几乎“喂到嘴边”。
    *   `stage1_init_y_min_m`, `max_m`: `[-0.05, 0.05]`
    *   `stage1_init_yaw_deg_min`, `max`: `[-2.0, 2.0]`
*   **奖励设置**：
    *   `paper_rg_dist_thresh`: `0.28` (物理可达的深度)
    *   `alpha_4 (rg)`: `50.0` (回归正常奖励，避免 Value Loss 爆炸)
    *   `alpha_5 (推盘惩罚)`: `3.0` (保持严格，防止养成坏习惯)
    *   `alpha_2, alpha_3 (姿态引导)`: `5.0` (全局姿态引导)
*   **预期结果**：在 1000~1500 代内，成功率达到 70% 以上，yaw < 3°。
*   **操作**：达到目标后，手动停止，保存 Checkpoint。

### 阶段二：Exp 7.2 - 微步泛化（中等范围）
**目标**：基于 7.1 的基础，开始扩展模型的泛化能力。
*   **起点**：加载 Exp 7.1 的最佳 Checkpoint。
*   **初始范围**：微步扩展。
    *   `stage1_init_y_min_m`, `max_m`: `[-0.1, 0.1]`
    *   `stage1_init_yaw_deg_min`, `max`: `[-5.0, 5.0]`
*   **奖励设置**：保持 7.1 的设置不变（继续用 5000 分锚定核心能力）。
*   **预期结果**：经历短暂震荡后，成功率稳定在 60% 以上，yaw < 5°。
*   **操作**：达到目标后，手动停止，保存 Checkpoint。

### 阶段三：Exp 7.3 - 全面泛化（宽范围 + 奖励平滑）
**目标**：挑战最终的泛化目标。
*   **起点**：加载 Exp 7.2 的最佳 Checkpoint。
*   **初始范围**：大幅扩展。
    *   `stage1_init_y_min_m`, `max_m`: `[-0.3, 0.3]` (甚至更宽)
    *   `stage1_init_yaw_deg_min`, `max`: `[-10.0, 10.0]`
*   **奖励设置**：
    *   如果模型在边缘位置崩溃，考虑将 `alpha_4` 从 5000 降至 1000 或 500，同时稍微降低 `alpha_5` (如降至 2.0)，给模型更多的探索空间，避免因为一次失误就被惩罚到崩溃。
*   **预期结果**：在宽范围内保持较高的成功率和良好的姿态。
