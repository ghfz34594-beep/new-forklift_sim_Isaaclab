## 3. 核心结论：RRL 范式全面胜出

通过这次严谨的对照实验，我们得出了本项目迄今为止最重要的结论：

1. **彻底废弃“坐标回归预训练”路线**：
   实验 1 花费了大量时间收集专家数据，并进行了复杂的坐标回归预训练，但最终的成功率（16.5%）与完全没有经过任务预训练的实验 2（16.3%）**完全一致**。这证明了试图让 CNN 去理解并输出物理坐标是一条费力不讨好的弯路。

2. **确立 RRL (ResNet as Representation for RL) 为主线**：
   实验 2 完美复刻了参考论文的核心思路。它仅仅使用了一个标准的、在 ImageNet 上预训练过的 ResNet18 模型，并且在 RL 训练过程中**全程冻结**了其权重。
   - **极高的效率**：它不仅省去了繁琐的数据收集和预训练步骤，而且在训练早期展现出了极其陡峭的学习曲线，用更少的代数追平了实验 1。
   - **特征极其稳定**：由于骨干网络被冻结，它完全避免了 RL 微调过程中常见的“灾难性遗忘”问题，为下游的 Actor/Critic 网络提供了极其稳定、高质量的通用视觉特征。

## 4. 瓶颈分析与下一步计划 (实验 3)

**当前瓶颈**：
无论使用哪种视觉特征，成功率都死死卡在 16.5%。这说明**视觉感知已经不再是当前的短板**。真正的瓶颈在于 **RL 的探索策略** 或 **Reward 函数的塑造**。智能体在接近目标时，缺乏足够的引导来完成最后的“精准插入”动作，导致其陷入了次优的探索陷阱。

**下一步计划 (实验 3)**：
1. **统一代码基座**：将代码库彻底切换到实验 2 的配置（ResNet18 + ImageNet 权重 + 全程冻结 `freeze_backbone=True`）。
2. **重塑 Reward 函数 (Reward Shaping)**：
   - 增加对齐后的“保持对齐”奖励，鼓励叉车在对准后稳定姿态。
   - 强化最后插入阶段（`insert_depth` > 0）的奖励梯度，引导智能体勇敢向前。
   - 调整对“推托盘”行为的惩罚力度，寻找探索与保守之间的最佳平衡点。
3. **集中算力**：不再并行运行，将所有系统资源集中用于跑这一个新实验，以加快迭代速度。
