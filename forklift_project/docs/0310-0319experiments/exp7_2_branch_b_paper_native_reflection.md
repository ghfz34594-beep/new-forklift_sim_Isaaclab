# Exp 7.2 (Branch B: Paper Native) 实验结果与深度反思

**日期**：2026-03-18
**实验分支**：`branch_b_paper_native`
**核心配置**：
- 消除过度设计：移除所有软门控、退火、空间衰减。
- 奖励函数：论文原生的绝对状态奖励 ($R^+ = \alpha_1 r_d + \alpha_2 r_{cd} + \alpha_3 r_{c\psi} + \alpha_4 r_g$)，使用 $\exp(-x/\sigma)$ 形式。
- 权重：$\alpha_1=5, \alpha_2=5, \alpha_3=5, \alpha_4=50$ (rg 降级为常规量级)。
- 参考轨迹：每 episode 动态生成 Bézier + 直线。
- 初始分布：宽广随机分布 (距离 1.5~2.5m, 横向 ±0.5m, 偏航 ±15°)。
- 视觉基座：ResNet34 (ImageNet 预训练，**全程冻结**)。

## 1. 实验结果现象

实验运行至约 Iteration 1500，呈现出极其典型且清晰的**两阶段演化**，最终陷入“原地冻结”的局部最优。

### 阶段一：暴力探索期 (Iter 0 ~ 450)
- **行为**：Agent 试图通过歪着车身（偏航角 30°~50°）强行撞击托盘来获取插入奖励。
- **数据**：`frac_inserted` 达到 15%~20%，但 `yaw_deg_mean` 极高，`pallet_disp_xy` (推盘位移) 高达 1.5m+。
- **分析**：这是在宽广初始分布下，Agent 找不到平滑插入路径时的本能反应。

### 阶段二：惩罚觉醒与原地冻结期 (Iter 450 ~ 1500+)
- **行为**：Agent 发现“暴力硬推”会吃到巨大的推盘惩罚 ($r_p$)，总体收益为负。于是它**彻底放弃了前进**，停在出生点附近，仅仅转动车身使车头对准参考轨迹。
- **数据**：
  - `frac_inserted` 降至 **0.0000**。
  - `pallet_disp_xy` 降至 **0.0000**。
  - `yaw_deg_mean` 奇迹般地收敛并稳定在 **6°~7°**。
  - `dist_front_mean` 停滞在 **1.76m** 左右（即初始出生距离）。
  - `rini` (停滞惩罚) 持续为 **-1.0**。
  - `Mean episode length` 极短（约 3.6 步），说明 Agent 出生后因不移动而迅速触发 early stop。

## 2. 深度反思与根因分析

Agent 为什么宁愿每步吃 $r_{ini}$ 惩罚，也不愿前进？

### 2.1 成功点：参考轨迹的 $r_{c\psi}$ 确实起效了
在 1.7m 的远距离，Agent 能够将偏航角从初始的随机状态（±15°）收敛到 6°~7°，这证明了**参考轨迹的切线引导 ($r_{c\psi}$) 是绝对有效的**。Agent 确实“看”到了轨迹的方向。

### 2.2 失败点：$r_d$ 的远场梯度消失 (The Flat Gradient Problem)
我们使用了指数形式的距离奖励：$r_d = \exp(-dist\_center / 1.0)$。
- 在 1.76m 处：$r_d = \exp(-1.76) \approx 0.172$
- 如果它前进 0.1m 到 1.66m 处：$r_d = \exp(-1.66) \approx 0.190$
- **边际收益**：前进 0.1m，单步奖励只增加 `(0.190 - 0.172) * 5.0 = 0.09` 分。

### 2.3 失败点：惩罚项的“威慑力”过大
前进不仅收益微薄，还伴随着巨大的风险：
- **动作突变惩罚 ($r_a$)**：要前进就必须踩油门，这会产生动作变化，导致 $r_a$ 扣分。
- **推盘恐惧 ($r_p$)**：之前阶段一的暴力探索让 Critic 网络对“靠近托盘”产生了深深的恐惧。
- **停滞惩罚 ($r_{ini}$) 的无力**：虽然停着不动会被扣 $\alpha_8 \times 1.0 = 5.0$ 分，但 Agent 算了一笔账：**“前进带来的微弱 $r_d$ 增益，根本不足以抵消踩油门带来的 $r_a$ 惩罚和未来可能撞击托盘的 $r_p$ 惩罚。”**

**结论**：在当前的权重配置下，**“原地摆正车头”是一个极其稳定的局部最优解 (Local Optima)**。Agent 找到了系统漏洞，实现了“躺平”。

## 3. 对《双线实验计划》的调整建议

本次实验完美印证了计划中“风险预判”环节提到的 **"0.3m 停车" (近场停滞) 和 "远距离梯度消失"**。

为了打破这个局部最优，我们需要在下一次迭代中调整奖励函数的平衡：

1. **增强前进动力 (Fixing Flat Gradient)**：
   - 方案 A：改回论文原生的 $1/x$ 形式（如 $1/(r_d + 0.1)$），它在远距离的梯度比 $\exp$ 更陡峭。
   - 方案 B：如果坚持用 $\exp$，必须大幅减小分母 $\sigma$（如从 1.0 降到 0.5），或者直接使用**差分势函数**（像 Master 分支那样，只奖励 `delta_dist`），让前进的每一步都有明确的即时正反馈。

2. **削弱“躺平”的舒适度**：
   - 大幅提高停滞惩罚的权重 $\alpha_8$（例如从 5.0 提高到 20.0），让“不动”的代价变得极其高昂，逼迫 Agent 必须走出舒适区。

3. **降低探索初期的动作惩罚**：
   - 降低 $r_a$ (动作突变) 的权重 $\alpha_7$。在 Agent 还没学会走路时，不要苛求它走得优雅。

本次实验证明了**参考轨迹方向引导的有效性**，我们只需解决**纵向推进动力不足**的问题。
