# 强化学习中的课程学习 (Curriculum Learning)

## 1. 课程学习的定义
课程学习（Curriculum Learning, CL）是机器学习（特别是强化学习）中的一种训练策略。它的核心思想**模仿人类的学习过程：从简单到复杂，循序渐进地掌握知识或技能**。

在强化学习的语境下，课程学习通常意味着：
*   **不一开始就让 Agent 挑战最终的、最困难的任务目标**。
*   而是先设计一系列**难度递增的中间任务（或放宽条件的任务）**。
*   Agent 先在简单的环境中学习基础技能，获得正反馈（Reward）。
*   随着 Agent 能力的提升，逐步收紧条件、增加难度，最终引导它完成原本不可能直接学会的复杂任务。

## 2. 为了解决什么问题才有了课程学习？

课程学习主要为了解决强化学习中的两大核心痛点：

### 痛点一：稀疏奖励下的“探索瘫痪” (Sparse Reward Problem)
*   **问题描述**：很多复杂任务（比如把叉车完美插入托盘、让机器人后空翻）只有在**最终成功的那一瞬间**才会给一个大奖（Sparse Reward）。在成功之前，无论 Agent 怎么努力，奖励都是 0。
*   **后果**：因为初始策略是完全随机的，Agent 在茫茫的动作空间中，可能随机探索几百万次都碰巧碰不到那个“成功状态”。它得不到任何正反馈，不知道往哪个方向努力，最终表现为“原地乱动”或“探索瘫痪”。
*   **课程学习的解法**：通过**降低初始门槛**（比如：只要叉车碰到托盘边缘就算成功，给奖励），让 Agent 很容易就能尝到甜头。它一旦学会了“走向托盘”，你再把门槛提高（要求插进一半），它就能在“走向托盘”的基础上继续探索，直到最终完全插入。

### 痛点二：复杂环境下的“局部最优陷阱” (Local Optima)
*   **问题描述**：为了缓解稀疏奖励，人们通常会设计密集的“引导奖励”（Shaping Reward，比如距离越近分越高）。但在极度复杂的任务中，如果直接上最高难度，Agent 很容易找到一些“投机取巧”的方法来骗取引导奖励，而不去完成最终任务。
    *   *例如我们的叉车：如果一开始就要求 0.1m 的极高精度插入，它发现自己怎么也插不进去，反而会因为撞击托盘被扣分。于是它干脆停在托盘门口，靠着“距离很近”和“姿态很正”白嫖 Shaping 奖励，死活不往前走。*
*   **课程学习的解法**：先在一个**容错率极高**的简单环境里（比如 0.4m 阈值，随便怎么歪都能插进去），让它明白“最终大奖（rg）的收益远远大于在门口蹭分”。等它建立了“必须插进去拿大奖”的坚定信念（Critic 网络收敛）后，再慢慢收紧姿态和距离要求。此时，它为了拿到那个已经印在脑子里的大奖，会主动去优化自己的微操，从而跨过局部最优。

## 3. 我们在叉车项目中是如何应用课程学习的？

我们在 `Exp 5.x` 系列实验中，完美地（也伴随着血泪地）实践了课程学习：

1.  **Stage 1 (破冰期 - Exp 5.4)**：
    *   **难度**：极低。`paper_rg_dist_thresh = 0.4m`（只要插进 2/3 就算成功）。
    *   **目的**：解决“探索瘫痪”。让随机乱开的叉车很容易就能触发 200 分的大奖，激活 Critic 网络，让它知道“往托盘里开是有肉吃的”。
2.  **Stage 2 (平滑退火 - Exp 5.5b)**：
    *   **难度**：中等。`paper_rg_dist_thresh = 0.3m`。
    *   **目的**：在它学会了“粗鲁地插”之后，稍微提高要求，逼迫它开始注意姿态，否则就会因为插不深而拿不到奖。
3.  **Stage 3 (终极挑战 - Exp 5.8)**：
    *   **难度**：逼近物理极限。`paper_rg_dist_thresh = 0.28m`。
    *   **目的**：在它已经能稳定完成 0.3m 的基础上，利用它已经建立的“必须拿大奖”的信念，配合极高的对齐奖励权重，逼迫它把偏航角压缩到 5° 以内，完成几乎完美的深插。

**反面教材（灾难性遗忘）**：
我们在 Exp 5.5 中，试图从 0.4m 直接跳到 0.2m。结果步子迈得太大，Agent 瞬间崩溃，完全忘记了怎么插入。这反证了课程学习中**“平滑过渡（退火）”**的极端重要性。
