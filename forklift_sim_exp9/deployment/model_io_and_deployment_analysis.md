## 0. 模型介绍：model_1999.pt

### 0.1 这个模型做什么

`model_1999.pt` 是一个 **叉车自动插盘举升** 的端到端控制策略。给定叉车与托盘之间的相对几何关系（15 维向量），模型每 33ms 输出一组 3 维动作指令（前进/后退、转向、升降），引导叉车完成以下完整操作序列：

```
 远场接近 → 航向对齐 → 横向对中 → 精准插入托盘口 → 叉齿深插 → 举升货物 → 稳定保持
```

整个过程无需任何人工干预或分阶段切换，策略自主完成从远距离接近到举升保持的全流程决策。

### 0.2 训练环境与配置

| 项目 | 值 |
|------|----|
| 仿真平台 | Isaac Lab (Isaac Sim 5.1) + PhysX GPU |
| 算法 | PPO (Proximal Policy Optimization, RSL-RL 实现) |
| 奖励版本 | **s1.0zB** — 三阶段势函数 + 里程碑 + 多层防作弊机制 |
| 并行环境数 | **1024** |
| 训练 iterations | **2000**（每 iteration 收集 64 steps × 1024 envs = 65,536 transitions） |
| 总训练 timesteps | **1.31 亿**（131,072,000） |
| 训练耗时 | **1 小时 43 分钟**（NVIDIA DGX，单 GPU） |
| Episode 时长 | 最长 36 秒（~1080 控制步），成功 episode 均长 ~450 步（~15 秒） |

### 0.3 模型架构

| 属性 | 值 |
|------|----|
| 模型文件 | `model_1999.pt`，**2.4 MB** |
| Actor 结构 | MLP `15 → [256, 256, 128] → 3`（ELU 激活） |
| Critic 结构 | MLP `15 → [256, 256, 128] → 1`（ELU 激活，部署时不用） |
| 输入归一化 | Running mean/std（训练时自适应，推理时冻结） |
| 输出裁剪 | [-1, 1] |
| 控制频率 | **30 Hz**（sim dt=1/120，decimation=4） |
| 动作噪声 std | **0.02**（训练末期已收敛到极低探索噪声） |

#### 核心指标

|------|----|------|

| 项目 | 表现 |
|------|------|

### 0.5 成功判定标准

模型需**同时**满足以下所有条件并**连续保持 10 个控制步（~0.33 秒）**：

| 条件 | 阈值 |
|------|------|
| 插入深度 | ≥ 40% 托盘深度（≥ 0.864 m） |
| 举升高度 | ≥ 0.3 m |
| 横向对齐 | ≤ 0.15 m（Schmitt exit: 0.18 m） |
| 偏航对齐 | ≤ 5°（Schmitt exit: 6°） |
| 叉齿尖对齐 | ≤ 0.12 m（近场额外约束） |

### 0.6 已知局限

| 局限 | 原因 | 影响 |
|------|------|------|
| 成功率未达 95%+ | 部分初始横偏 >0.4m 的 case 对齐缓慢导致超时 | 约 11% 的 episode 失败 |
| 托盘被推动 | 低插入度下叉齿碰撞推移托盘 | 个别 case 位移达 2.3m |
| 固定托盘位姿 | 训练时托盘始终在原点、无旋转 | sim2real 需验证泛化性 |
| 无视觉输入 | 依赖精确几何状态向量 | 部署需传感器管线提供观测 |

## 2. 输入向量：15 维观测

### 2.1 逐维度说明

| 索引 | 维数 | 名称 | 物理含义 | 值域 | 坐标系 |
|------|------|------|----------|------|--------|
| 0–1 | 2 | `d_xy_r` | 托盘相对叉车的平面位置 | 约 [-5, 5] m | **robot frame**（x=前, y=左） |
| 2 | 1 | `cos_dyaw` | 偏航角差余弦 | [-1, 1] | — |
| 3 | 1 | `sin_dyaw` | 偏航角差正弦 | [-1, 1] | — |
| 4–5 | 2 | `v_xy_r` | 叉车平面线速度 | 约 [-3, 3] m/s | **robot frame** |
| 6 | 1 | `yaw_rate` | 叉车偏航角速度 | 约 [-2, 2] rad/s | world Z 轴 |
| 7 | 1 | `lift_pos` | 升降关节位置 | [0, ~2] m | 关节空间 |
| 8 | 1 | `lift_vel` | 升降关节速度 | 约 [-1, 1] m/s | 关节空间 |
| 9 | 1 | `insert_norm` | 归一化插入深度 | [0, 1] | 托盘局部轴投影 |
| 10–12 | 3 | `prev_actions` | 上一步动作（drive, steer, lift） | [-1, 1] | — |
| 13 | 1 | `y_err_obs` | 横向误差（带符号，归一化） | [-1, 1] | **pallet center-line frame** |
| 14 | 1 | `yaw_err_obs` | 偏航误差（带符号，归一化） | [-1, 1] | **pallet center-line frame** |

### 2.2 按获取难度分类

```
┌────────────────────────────────────────────────────────────────────┐
│  Easy（车载传感器可直读）                                           │
│  ──────────────────────                                            │
│  [4-5]  v_xy_r        ← 轮速计/IMU                                │
│  [6]    yaw_rate      ← IMU 陀螺仪                                │
│  [7]    lift_pos      ← 升降编码器                                  │
│  [8]    lift_vel      ← 升降编码器差分                              │
│  [10-12] prev_actions ← 控制器自身缓存                              │
├────────────────────────────────────────────────────────────────────┤
│  Medium（需外部感知，但维度低、精度可控）                              │
│  ──────────────────────                                            │
│  [0-1]  d_xy_r        ← 需要感知托盘位置并变换到 robot frame          │
│  [2-3]  cos/sin_dyaw  ← 需要感知托盘朝向                            │
│  [13]   y_err_obs     ← 需要感知托盘 center-line 相对横向偏差         │
│  [14]   yaw_err_obs   ← 需要感知托盘 center-line 相对偏航偏差         │
├────────────────────────────────────────────────────────────────────┤
│  Hard（需精确几何推理）                                              │
│  ──────────────────────                                            │
│  [9]    insert_norm   ← 叉齿尖端已插入托盘的深度（归一化）             │
│                         需要精确知道 tip 位置 + 托盘开口位置           │
└────────────────────────────────────────────────────────────────────┘
```

**关键发现**：15 维输入中，8 维（indices 4–8, 10–12）可由车载传感器直读；6 维（indices 0–3, 13–14）本质上都是「托盘位姿」的不同投影（只需知道托盘位置 + 朝向即可全部算出）；剩余 1 维（index 9）需要额外测量插入深度。

换言之，**部署时真正需要感知系统提供的核心信息只有两项**：

1. **托盘 6-DoF 位姿**（位置 + 朝向） → 可推导 d_xy_r, cos/sin_dyaw, y_err_obs, yaw_err_obs
2. **叉齿插入深度** → insert_norm

## 3. 输出向量：3 维动作

| 索引 | 名称 | 含义 | 网络输出 | 映射到物理量 |
|------|------|------|----------|-------------|
| 0 | `drive` | 前进/后退 | [-1, 1] | ×20 → 车轮角速度 ±20 rad/s |
| 1 | `steer` | 左/右转向 | [-1, 1] | ×0.6 → 转向角 ±0.6 rad (~34°) |
| 2 | `lift` | 货叉升/降 | [-1, 1] | ×0.5 → 升降速度 ±0.5 m/s |

部署时需将 [-1, 1] 输出映射为实车 CAN 指令（车速、转角、液压阀开度）。

## 4. 两条部署路径对比

### 路径 A：传感器管线 + 现有网络（Sensor Pipeline）

```
  双目/深度相机                      IMU + 编码器
       │                                │
       ▼                                ▼
  托盘检测 + 位姿估计              v_xy_r, yaw_rate,
  (6-DoF pose)                    lift_pos, lift_vel
       │                                │
       ├── d_xy_r (旋转到 robot frame)   │
       ├── cos/sin_dyaw                  │
       ├── y_err_obs (投影 + 归一化)      │
       ├── yaw_err_obs (投影 + 归一化)    │
       │                                │
       │    ToF / 激光测距                │
       │         │                       │
       │    insert_norm                  │
       │         │                       │
       ▼         ▼                       ▼
  ┌──────────────────────────────────────────┐
  │  现有 MLP Actor（15→3，2.4 MB，30 Hz）    │
  │  直接部署，无需重新训练                     │
  └──────────────────────────────────────────┘
       │
       ▼
   drive / steer / lift → CAN 总线
```

**各维度传感器方案**：

| 观测维度 | 传感器方案 | 精度要求 | 成熟度 |
|----------|-----------|---------|--------|
| d_xy_r (0–1) | 双目相机 + 托盘检测 / 深度相机 + ICP | ±5 cm | 高（工业成熟） |
| cos/sin_dyaw (2–3) | 同上（从 6-DoF 位姿提取） | ±2° | 高 |
| v_xy_r (4–5) | 轮速计 + IMU 融合 | ±0.05 m/s | 很高（标配） |
| yaw_rate (6) | IMU 陀螺仪 | ±0.01 rad/s | 很高（标配） |
| lift_pos (7) | 升降编码器 / 拉绳位移传感器 | ±1 mm | 很高（标配） |
| lift_vel (8) | 编码器差分 | ±5 mm/s | 很高 |
| insert_norm (9) | **ToF 传感器 / 1D 激光测距** 装在叉齿上 | ±1 cm | 中（需定制安装） |
| prev_actions (10–12) | 控制器内部缓存 | 精确 | — |
| y_err_obs (13) | 同 d_xy_r（数学投影） | ±3 cm | 高 |
| yaw_err_obs (14) | 同 cos/sin_dyaw（数学投影） | ±1° | 高 |

**优势**：
- 模型已训练完成，**可直接部署**，无需额外训练
- MLP 推理极轻（2.4 MB，< 0.1 ms @ 30 Hz），可在任何边缘设备运行
- 每个维度的物理含义明确，**可独立验证和调试**
- sim2real gap 仅来自传感器噪声，可通过 domain randomization 缓解
- 传感器管线各模块可独立替换/升级

**劣势**：
- 需要精确的托盘检测与 6-DoF 位姿估计（核心依赖）
- insert_norm 需额外安装测距传感器
- 管线延迟（检测 + 估计 + 推理）需控制在 33 ms 以内
- 托盘遮挡/反光等极端场景可能导致感知失败

---

### 路径 B：端到端视觉策略（Vision End-to-End）

```
  RGB / RGB-D 相机
       │
       ▼
  ┌───────────────────────────────────────┐
  │  视觉编码器（CNN/ViT/ResNet）           │
  │  + 策略头（MLP）                        │
  │  image → latent → 3D action            │
  │  需要重新训练（仿真中渲染 + RL）          │
  └───────────────────────────────────────┘
       │
       ▼
   drive / steer / lift → CAN 总线
```

**优势**：
- 不依赖手工感知管线，理论上更泛化
- 不需要单独的 insert_norm 测距传感器
- 长期可能更易扩展到新托盘类型

**劣势**：
- **需要从零重新训练**：当前 15 维 MLP 策略无法复用
- 仿真渲染质量直接决定 sim2real gap（Isaac Sim 渲染可用但需大量调参）
- 训练成本剧增：视觉编码器 + RL 联合训练需要 GPU 集群，收敛时间以天/周计
- 模型体积从 2.4 MB 膨胀到 50–200 MB，推理延迟从 < 1 ms 到 10–30 ms
- 调试困难：无法分维度定位「是感知错了还是决策错了」
- 真实世界的光照、反光、遮挡等视觉鲁棒性是已知难题

## 5. 综合评估矩阵

| 评估维度 | 路径 A（传感器管线） | 路径 B（端到端视觉） |
|----------|:---:|:---:|
| 能否复用现有模型 | **是（直接部署）** | 否（需重训） |
| 额外训练成本 | 0（仅需标定） | 高（数天~数周 GPU） |
| 硬件成本 | 中（双目相机 + ToF） | 低（仅 RGB 相机） |

| 可调试性 | **高（逐维度检查）** | 低（黑盒） |
| sim2real gap | **小（仅传感器噪声）** | 大（渲染真实感） |
| 推理延迟 | **< 5 ms** | 10–30 ms |
| 部署设备要求 | 任意 ARM/x86 | 需 GPU（Jetson+） |
| 长期扩展性 | 中 | 高 |
| 落地时间 | **2–4 周** | 2–6 月 |

## 6. 建议

### 短期（首次实车验证）：路径 A

理由：
1. **现有模型可直接使用**，最核心的对齐、插入、举升策略已在仿真中验证
2. 15 维观测中 8 维已由标配传感器覆盖，仅需解决两个感知子问题：
   - **托盘 6-DoF 位姿**：双目/深度相机 + 托盘检测（工业成熟方案）
   - **插入深度**：叉齿上安装 ToF 传感器（简单可靠）
3. 可以在实车上**逐步验证每一个维度的精度**，出问题时能精确定位
4. 预计 2–4 周内可完成首次实车闭环测试

### 中期（产品化）：路径 A + Domain Randomization

- 在仿真中对 15 维观测添加噪声（模拟传感器误差），微调策略鲁棒性
- 对关键维度（d_xy_r, insert_norm）做 ±10% 噪声注入
- 可选：用 Teacher-Student 蒸馏进一步提升泛化

### 远期（可选探索）：路径 B

- 待路径 A 在实车上稳定运行后，可并行探索端到端视觉方案
- 使用路径 A 的运行数据作为 expert demonstration 进行模仿学习
- 或用当前 MLP 策略作为 teacher，视觉策略作为 student 做蒸馏

## 7. 路径 A 具体落地清单

### 7.1 感知模块

```
传感器              → 输出               → 对应观测维度
─────────────────────────────────────────────────────
双目相机/深度相机    → 托盘 6-DoF 位姿    → [0-3], [13-14]
IMU                → 角速度/加速度       → [4-6]
轮速编码器          → 车体速度           → [4-5]
升降编码器          → 关节位置/速度       → [7-8]
ToF (叉齿安装)      → 插入深度           → [9]
控制器缓存          → 上一步动作         → [10-12]
```

### 7.2 坐标变换

从双目相机得到 `pallet_pos_cam` 和 `pallet_yaw_cam` 后，需要：

```python
# 相机外参标定后：camera frame → robot frame
pallet_pos_robot = R_cam2robot @ pallet_pos_cam + t_cam2robot

# 计算观测维度
d_xy_r = pallet_pos_robot[:2]                           # obs[0:2]
dyaw = pallet_yaw_robot - robot_yaw                     # 
cos_dyaw, sin_dyaw = cos(dyaw), sin(dyaw)              # obs[2:4]

# center-line frame 投影
y_signed = dot(robot_pos - pallet_pos, pallet_lateral_axis)
y_err_obs = clip(y_signed / 0.8, -1, 1)                # obs[13]
yaw_err_obs = clip(dyaw / deg2rad(15), -1, 1)          # obs[14]
```

### 7.3 insert_norm 获取方案

| 方案 | 传感器 | 安装位置 | 测量原理 |
|------|--------|---------|---------|
| A | ToF 传感器 | 叉齿尖端朝前 | 测量 tip 到托盘后挡板距离 → 推算插入深度 |
| B | 1D 激光测距 | 叉齿根部朝前 | 测量到托盘开口面距离 → 结合叉齿长度推算 |
| C | 双目视觉推算 | 相机 | 从 6-DoF 位姿 + 叉车运动学推算（精度较低） |

推荐方案 A（ToF），成本低、精度高、无遮挡问题。

# 每步推理（30 Hz）
def infer(obs_15d: np.ndarray) -> np.ndarray:
    """
    Args:
        obs_15d: shape (15,), 按文档 Section 2.1 顺序拼接
    Returns:
        action: shape (3,), [drive, steer, lift] in [-1, 1]
    """
    obs = (obs_15d - obs_mean) / sqrt(obs_var + 1e-8)  # 归一化
    action = actor_forward(obs)                          # MLP 前向
    action = np.clip(action, -1.0, 1.0)                 # 裁剪
    return action

steer_cmd = action[1] * 0.6       # rad   → 转向角 CAN 指令  
lift_cmd  = action[2] * 0.5       # m/s   → 液压阀开度 CAN 指令
