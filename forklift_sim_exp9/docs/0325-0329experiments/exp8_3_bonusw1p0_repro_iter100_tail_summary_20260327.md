## 2. 一页结论

- `seed42` 仍然是这组里最稳的 seed，`r1` 和 `r2` 两次都能在尾段保持非零 `push_free / hold`，其中 `r1_seed42` 最好。
- `seed43` 的问题依旧最典型:
  - `r1_seed43` 变成“大量 dirty insert 但没有 hold/success”
  - `r2_seed43` 直接塌成“几乎不插”
- `seed44` 介于两者之间:
  - `r1_seed44` 有一定插入和少量 clean 信号，但没有尾段 success
  - `r2_seed44` 直接塌成不插
- 两个 repeat 呈现出明显不同风格:
  - `r1`: 更激进，更容易插进去，但 dirty insert 也更高
  - `r2`: 更保守，推盘很小，但大量 case 直接不插

一句话总结:

**`bonusw=1.0` 在 `100 iter` 下仍然没有解决“要么脏插、要么不插”的复现方差；它能稳定工作主要还是集中在 `seed42`。**

## 7. 关键发现

### 7.1 `100 iter` 没有自动解决复现方差

如果 `100 iter` 足以消除方差，我们应该看到:

- `r1/r2` 差异缩小
- `seed43/44` 更稳定地进入 clean insert

但结果并没有这样发展。  
反而是:

- `seed43` 继续在“脏插很多”和“不插”之间摇摆
- `seed44` 继续在“部分插入但不够 clean”和“不插”之间摇摆

所以把迭代数从 `50` 提到 `100`，并没有自动把系统推到稳定 regime。

### 7.2 `seed42` 是唯一相对可重复的正例

`seed42` 的两次 repeat 都不是完美，但都具备:

- 非零 `push_free`
- 非零 `hold`
- 非零 `r_clean_insert_bonus`
- 很低的 `pallet_disp_xy_mean`

这说明当前配置并非完全不可行。  
问题不是“这套 reward 永远不工作”，而是“它只在部分随机轨迹下稳定进入正确 attractor”。

### 7.3 `seed43` 的主要失败模式仍是 dirty insert 锁死

`r1_seed43` 的尾窗特征非常典型:

- `inserted = dirty_insert = 0.5938`
- `push_free = hold = success = 0`
- `pallet_disp_xy_mean = 0.8702`
- `d_traj_mean = 1.0048`

这说明它在后段不是“不会靠近”，而是“反复插入，但始终以 dirty insert 形式存在”。

### 7.4 `r2_seed43/44` 是典型的保守塌缩

这两个 run 的共同特征是:

- `inserted = 0`
- `bonus = 0`
- `hold = 0`
- `disp` 极小

这不是“更干净的成功”，而是“基本不做尝试”。

---

## 8. 当前最合理的判断

这组 `100 iter` 复现实验说明:

1. `bonusw=1.0` 的方向仍然是活的，因为 `seed42` 可以重复打出 clean 信号。
2. 但它还远远称不上稳定配置，因为 `seed43/44` 仍然高度依赖随机轨迹。
3. 当前最主要的问题已经不是“尾段 bonus 是否够强”，而是:
   - 训练早中期是否会滑进 `dirty insert attractor`
   - 或者直接滑进 `保守不插 attractor`

---

## 9. 一句话判断

**这 6 条 `100 iter` 复现实验表明，`bonusw=1.0` 在 `seed42` 上能重复出现 clean insert 信号，但整体仍然卡在“要么脏插、要么不插”的双峰失败模式里；单纯把迭代数加到 `100` 还不足以解决复现方差。**
