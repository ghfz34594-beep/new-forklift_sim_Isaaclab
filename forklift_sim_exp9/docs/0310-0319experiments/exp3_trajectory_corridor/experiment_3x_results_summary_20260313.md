# 实验 3.x 阶段性结果总结 (3.1 - 3.3)

## 1. 实验 3.1：参考轨迹走廊 (Trajectory-lite)
**目标**：解决智能体在远场缺乏明确接近路径，导致“不敢进门”或“乱走”的问题。
**方法**：引入基于 Bézier 曲线的参考轨迹，用 `phi_traj` 替代原有的距离带奖励 `phi1`。
**结果**：
- **成功点**：智能体学会了完美跟随轨迹，`traj/yaw_traj_deg_mean` 降至 6° 左右，`diag/pallet_disp_xy_mean` 降至 0.02m（几乎不推托盘）。
- **失败点**：陷入了新的局部最优——“乖宝宝综合征”。`phase/frac_inserted` 彻底降为 0。智能体发现只要停在门外不进去，就能一直白嫖轨迹居中奖励，且绝对不会触发推盘惩罚。
- **结论**：3.1 成功把车带到了门前，但缺乏“临门一脚”的动力。必须引入近场 Commit 奖励。

## 2. 实验 3.2：近场 Commit 奖励
**目标**：解决 3.1 中智能体在门前停滞不前的问题。
**方法**：引入近场门控 `gate_commit`（距离<1.0m 且对准时打开），并增加 `r_commit_front`（鼓励靠近）和 `r_commit_insert`（鼓励插入）。
**结果**：
- **成功点**：`phase/frac_inserted` 成功破零，最高飙升至 19.5%！无碰撞成功率也回升至 12.5% 左右。智能体终于敢于把叉子插进去了。
- **失败点**：老问题“推土机”强势回归且愈演愈烈。`diag/pallet_disp_xy_mean` 暴涨至 2.0m 以上，偏航角飙升至 50°+。
- **原因分析**：智能体发现了 Reward Hacking（奖励漏洞）。它故意把方向打歪，斜着浅插进托盘，然后顶着托盘满场跑。这样既能持续吃到 `r_commit_insert`，又能因为是浅插而少扣推盘惩罚。
- **结论**：3.2 成功激发了插入意愿，但暴露了推盘惩罚机制的漏洞。必须重构推盘惩罚。

## 3. 实验 3.3：条件化推盘惩罚
**目标**：解决 3.2 中智能体为了吃插入奖励而变身“推土机”的问题。
**方法**：将推盘惩罚改为 `pen_push_cond`。远场重罚（1.0），近场轻罚（0.35 允许探索），死区加倍重罚（+0.6 严打插歪了还推）。
**结果**：
- **初期（<100代）**：推土机行为被彻底镇压，托盘位移降至 0.006m，偏航角恢复到 7°。但插入率也随之跌回 0。
- **中后期（>1000代）**：随着训练深入，智能体再次找到了平衡点。插入率恢复到 12% 左右，但托盘位移也回升到了 1.0m ~ 1.4m，偏航角在 24° ~ 40° 之间。
- **结论**：3.3 确实增加了推盘的成本，让智能体收敛了一些，但依然没有堵住“斜着浅插推着走”的漏洞。只要门控在死区依然打开，它就会觉得“推盘扣的分没有插入给的多”。

## 4. 当前核心矛盾与下一步计划 (实验 3.4)
目前的根本问题在于：**近场门控 (`gate_commit`) 的关闭条件不够严格**。
当智能体插歪了（进入死区）时，它依然能拿到 `r_commit_insert`。

**实验 3.4 计划：死区撤退与门控熔断**
1. **门控熔断**：修改 `gate_commit` 的逻辑，一旦检测到进入死区（`insert_norm > thresh` 且 `y_err > thresh`），门控必须**强制归零**。切断它在死区白嫖插入奖励的途径。
2. **撤退奖励 (`r_escape`)**：当它处于死区时，如果它主动减小 `insert_norm`（把叉子拔出来），给予奖励，鼓励它退出来重新对准。
