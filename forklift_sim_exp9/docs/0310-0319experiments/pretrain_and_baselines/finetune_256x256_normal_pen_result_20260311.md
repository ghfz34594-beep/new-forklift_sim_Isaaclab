# 256x256 分辨率预训练+微调 (正常惩罚) 实验结论 (2026-03-11)

## 1. 实验背景与目标

在上一轮实验中，我们发现“强推盘惩罚 (权重 3.0)”会导致 Agent 变得极端保守（宁可不插也不推盘）。
因此，本轮实验将推盘惩罚回调至正常水平（权重 1.0，恢复浅插入豁免），旨在观察：**在没有过度惩罚的束缚下，256x256 的预训练特征能否引导 Agent 学会精准插入？**

## 2. 实验配置

- **分支**：`exp/vision_cnn/finetune_256x256`
- **训练日志**：`logs/20260311_151217_train_finetune_256x256_normal_pen_env64.log`
- **训练规模**：`num_envs=64`
- **视觉输入**：`256x256`，60 度俯视相机
- **Backbone**：加载 `outputs/vision_pretrain_256x256/best_backbone.pt`
- **冻结策略**：前 500 个 policy iterations 冻结 Backbone
- **奖励函数**：正常推盘惩罚（`k_pallet_push_pen = 1.0`, `pallet_push_insert_gate = 0.15`）

## 3. 实验过程与结果

该实验运行至 **Learning iteration 133** 时被手动终止。

### 3.1 核心指标变化趋势

| Iteration | `success_rate_total` | `frac_inserted` | `pallet_disp_xy_mean` (推盘位移) | `err/lateral_near_success` (横向误差) |
|---|---|---|---|---|

### 3.2 现象深度剖析 (The "Aha!" Moment)

本轮实验揭示了一个非常底层的逻辑闭环：

1. **“假成功”与“推土机”重现**：
   - 成功率很快稳定在 22%~25% 左右，但这是一种**“假成功”**。
   - 托盘平均位移（`pallet_disp_xy_mean`）高达 2.0 米！这意味着 Agent 根本没有精准对齐，而是直接撞上托盘，一路往前推，靠着暴力挤压偶尔触发了“插入 > 0.4m”的成功条件。
   - 接近成功时的横向误差（`lateral_near_success`）高达 0.25m ~ 0.30m，偏航误差高达 10° 以上。

2. **为什么它对不准？**
   - 我们回顾了 256x256 预训练阶段的最佳模型指标：`Val MAE -> X: 0.369m, Y: 0.159m, Yaw: 3.49°`。
   - 预训练 Backbone 的**内生横向误差 (Y) 是 0.16 米 (16 厘米)**！
   - 托盘插孔的物理容差极小。如果视觉特征本身就带有 16 厘米的误差，Agent 在冻结 Backbone 的情况下，**绝对不可能**做到不碰托盘的完美插入。

3. **冻结策略 (Freeze) 成为了最大阻碍**：
   - 我们设置了 `freeze_backbone_updates = 500`。
   - 在这 500 代里，Actor 被迫使用一个“带有 16cm 误差的模糊眼睛”来学习控制策略。
   - **在正常惩罚下**（本轮实验）：Actor 发现既然看不准，那就直接往前撞，反正推盘惩罚不大，撞进去就能拿到 `r_pot` 奖励。于是它收敛到了“推土机”局部最优。
   - **在强惩罚下**（上一轮实验）：Actor 发现往前撞会被重罚，但它又看不清到底该怎么微调，于是它收敛到了“原地不动、放弃插入”的局部最优。

## 4. 结论

**结论：预训练特征的精度（16cm 误差）不足以支撑零碰撞的完美插入。长时间冻结 Backbone（500代）会导致 RL 控制策略在“模糊视觉”下过早收敛到局部最优（要么推盘，要么放弃）。**

监督学习（预训练）只能学到“大概位置”，真正像素级的“精细对齐”必须依赖 RL 端到端（End-to-End）的梯度回传。

## 5. 下一步行动

我们必须让 RL 尽早接管视觉特征的微调！

**下一步计划：取消或大幅缩短 Backbone 冻结期**
1. 保持 256x256 分辨率和预训练权重加载。
2. 保持正常推盘惩罚（权重 1.0）。
3. **修改冻结策略**：
   - 方案 A：完全不冻结（`freeze_backbone = False`），从第 0 代开始端到端联合训练。
   - 方案 B：极短冻结（`freeze_backbone_updates = 50`），只让 Actor 的 MLP 适应一下预训练特征的分布，然后立刻解冻。
4. 建议采用**方案 B (冻结 50 代)**，既能防止初期随机 Actor 梯度破坏 Backbone，又能尽早让 RL 修正视觉误差。
