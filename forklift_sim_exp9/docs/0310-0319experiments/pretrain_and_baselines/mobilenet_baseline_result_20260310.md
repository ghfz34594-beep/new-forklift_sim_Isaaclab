# MobileNetV3 视觉 Baseline 训练结果分析 (2026-03-10)

## 1. 实验概况
- **实验目标**：验证将视觉 Backbone 升级为 MobileNetV3-Small 后，端到端强化学习（从零开始训练）在 60 度俯视视角和远距离初始化条件下的表现。
- **分支**：`exp/vision_cnn/cnn_pretrain`
- **训练步数**：2000 iterations (约 3276 万 timesteps)
- **耗时**：约 6 小时 44 分钟 (单次 iteration 耗时约 11.7s ~ 13.0s)

## 2. 核心指标分析

### 2.1 成功率 (Success Rate)
- **最终总成功率 (`episode/success_rate_total`)**：**20.29%**
- **近期成功率 (`episode/success_rate_ema`)**：**21.73%**
- **接近成功率 (`diag/near_success_frac`)**：**23.05%**

**趋势分析**：
- 前 500 个 iteration：成功率快速上升，从 0% 迅速攀升至 19.55%。
- 第 500 ~ 2000 个 iteration：成功率进入了漫长的平台期（Plateau），在 19.5% ~ 20.3% 之间缓慢波动，最终收敛在 20% 左右。
- **对比**：相比于之前基础 3 层 CNN（成功率不到 5%），MobileNetV3 展现出了强大的特征提取能力，让网络在早期就能学到粗略的对齐策略。

### 2.2 对齐精度与误差 (Alignment Errors)
虽然有 20% 的成功率，但分析失败的 case，主要卡在对齐精度不够：
- **接近成功时的偏航角误差 (`err/yaw_deg_near_success`)**：最终稳定在 **9° ~ 11°** 左右。
- **接近成功时的横向误差 (`err/lateral_near_success`)**：最终稳定在 **0.22m ~ 0.30m** 左右。

*注：当前课程的成功阈值为横向误差 < 0.15m，偏航角误差 < 8.0°。网络输出的动作精度一直在阈值边缘徘徊，导致很难稳定保持 (Hold) 足够的时间来触发成功。*

### 2.3 物理交互现象 (Physical Interaction)
- **托盘位移 (`diag/pallet_disp_xy_mean`)**：平均达到了 **1.7m ~ 2.1m**，最大位移甚至超过 6m。
- **现象解释**：由于对齐精度不够（横向和角度有偏差），货叉经常没有完美对准插孔，而是顶到了托盘的边缘或中间的木档。在持续向前的动作下，叉车变成了“推土机”，推着托盘在场地上移动。

## 3. 结论与下一步计划 (Next Steps)

### 3.1 结论
1. **Backbone 升级有效**：MobileNetV3 显著优于基础 CNN，证明了更深、更复杂的网络结构对于视觉 RL 任务是必要的。
2. **纯 RL 从零训练 (Scratch) 存在精度瓶颈**：在远距离初始化下，仅靠稀疏的 RL 奖励信号，网络很难学到精确的毫米级/度级对齐特征。20% 的成功率似乎是当前架构和训练方式的极限。

### 3.2 下一步计划
为了突破 20% 的成功率瓶颈，解决“推盘”和“对齐不准”的问题，必须引入**视觉预训练 (Visual Pretraining)**：

1. **收集数据集**：使用专家策略（`master` 分支的低维状态模型）在环境中跑数据，收集 `(图像, 真实物理状态)` 对。
2. **预训练 MobileNetV3**：
   - 方案 A（位姿预测）：使用监督学习，让 MobileNetV3 预测托盘的相对位置 (x, y) 和偏航角 (yaw)。
   - 方案 B（对比学习）：使用自监督方法提取空间特征。
3. **RL 微调 (Finetune)**：将预训练好的 MobileNetV3 权重加载到 `VisionActorCritic` 中（利用当前分支已写好的 `load_backbone_checkpoint` 逻辑），冻结或以极小的学习率微调 Backbone，让 PPO 专注于学习控制策略。
