## 一、 实验背景
在 Exp 5.9 中，我们通过“核弹级奖励+早停”获得了在严苛初始条件下（横向 ±5cm，偏航 ±2°）的完美插入模型。但为了实现 Sim2Real，Agent 必须具备从更宽广的初始位置自主寻的并调整姿态的能力。
Exp 6.0 的目标是**打破“温室”，恢复泛化能力**。

## 三、 探讨与方案设计

如果我们要**真正增加难度**，应该将范围扩大。例如：
*   **方案 A（温和扩展）**：横向 `[-0.2m, 0.2m]`，偏航 `[-5.0°, 5.0°]`。
*   **方案 B（中度扩展）**：横向 `[-0.5m, 0.5m]`，偏航 `[-10.0°, 10.0°]`。

**奖励函数的潜在问题预警**：
目前我们的 `r_lat = exp(-tip_y_err / 0.2)` 和 `r_yaw = exp(-yaw_err_rad / 0.2)` 是**全局生效**的。
这意味着，如果出生在很远的地方（比如 3 米外），它可能会选择**原地打转**把偏航角对准，或者**横向平移**把横向误差消除，而不是一边往前开一边调整。
更糟糕的是，如果它出生在偏航角很大的地方，它可能会发现“倒车”或者“原地摆 Pose”能更快地拿到 `r_lat` 和 `r_yaw` 的奖励，从而再次陷入**局部最优（摆 Pose 刷分）**。

**解决方案（空间衰减 Shaping）**：
为了防止远距离“摆 Pose”，我们应该让姿态奖励（`r_lat`, `r_yaw`）的权重随着距离的缩短而增加。
距离远时，主要靠 `r_dist` 引导它靠近；距离近时，`r_lat` 和 `r_yaw` 才开始发挥主导作用，逼迫它对准。

修改 `env.py` 中的奖励计算：
```python
# 引入空间衰减因子 (距离越近，decay_factor 越接近 1；距离远时接近 0)
# 假设 3 米外几乎没有姿态奖励，1 米内姿态奖励拉满
decay_factor = torch.exp(-dist_center / 1.0) 

# 动态调整 Shaping 权重
R_plus = (
    self.cfg.alpha_1 * r_dist +
    self.cfg.alpha_2 * r_lat * decay_factor +  # 姿态奖励乘以衰减因子
    self.cfg.alpha_3 * r_yaw * decay_factor +  # 姿态奖励乘以衰减因子
    self.cfg.alpha_4 * rg +
    r_lift
)
```

## 五、 Exp 6.0 失败与 6.0b 修正 (2026-03-15 更新)
**Exp 6.0 执行结果**：遭遇**灾难性遗忘**。在放宽到 `±0.5m` 和 `±15°` 后，成功率迅速跌至 0%，偏航角误差飙升至 40°。
**原因分析**：
1. 步子太大，探索空间爆炸。
2. 引入的“空间衰减 Shaping”导致 Agent 在远处得不到姿态奖励，盲目靠近后姿态已无法挽救。
3. 大奖降回 200 分，不足以在巨大的探索噪声中产生足够的正向刺激。

**Exp 6.0b 修正方案 (微步扩展)**：
基于上述反思，我们立即停止了 6.0，并启动 6.0b：
1. **起点**：退回 5.9 的 `model_836.pt`。
2. **微步放开**：横向仅放宽至 `[-0.1m, 0.1m]`，偏航角仅放宽至 `[-5.0°, 5.0°]`。
3. **保持核弹奖励**：`alpha_4 = 5000`，用重赏对抗遗忘。
4. **恢复全局 Shaping**：移除空间衰减，逼迫它在远处就开始调整姿态。

**6.0b 初期表现**：策略奏效。成功率在起步短暂下跌后，迅速恢复并稳定在 **40%** 左右，偏航角控制在 10° 以内。Agent 正在这个“舒适区边缘”重新学习和收敛。
