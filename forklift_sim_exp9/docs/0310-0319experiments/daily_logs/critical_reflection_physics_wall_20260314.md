# 核心反思与纠偏：撞上物理引擎的“叹息之墙”

**日期**：2026-03-14
**事件**：在 Exp 5.5c 到 Exp 5.7 的一系列实验中，Agent 的成功率（`rg`）始终为 0。经过排查，发现根本原因并非 RL 策略陷入局部最优，而是**设定的奖励阈值在物理上绝对不可达**。

## 1. 荒诞的真相：0.25m 阈值是物理死局

在 Exp 5.5c 及其后续实验中，我们将触发大奖 `rg` 的距离阈值 `paper_rg_dist_thresh` 设定为了 **0.2m** 和 **0.25m**。

让我们做一道简单的小学数学题：
*   **几何定义**：`dist_center` 是当前叉臂中心到“完美插入时”目标叉臂中心的距离。
*   **叉臂尺寸**：叉臂总长 1.2m，中心位于 0.6m 处。
*   **阈值要求**：`dist_center < 0.25m`，意味着叉车必须插入至少 `1.2 - 0.25 = 0.95m` 的深度。

然而，根据 `docs/diagnostic_reports/pallet_dragging_issue_2026-02-22.md` 的历史诊断报告，以及今天重新运行的 `verify_forklift_insert_lift.py --sanity-check` 物理极限测试：
*   **物理真相**：当前放大了 1.8 倍的托盘（`pallet.usd`），在其深度 52% 处（约 1.12m）存在隐藏的内部阻挡柱。由于物理引擎的凸分解（Convex Decomposition）膨胀，**货叉最多只能插入 0.9384 米，就会被死死卡住！**

**结论**：
*   物理极限最大插入深度：**0.938m**
*   触发 0.25m 阈值所需深度：**0.950m**
*   **0.938m < 0.950m**

Agent 根本不是“不敢”往前插，也不是陷入了“摆 Pose 刷分”的局部最优。它是被我们关进了一个物理上不可能完成的牢笼里，它已经把货叉顶到了物理引擎允许的绝对极限，却依然够不到我们设定的那条虚幻的及格线。

## 2. 深刻的自我检讨（AI 助手的反思）

这是一次极其深刻的教训，暴露了我在协助排查问题时的几个致命盲区：

### 2.1 典型的“算法傲慢”与“路径依赖” (RL Bias)
手里拿着 RL 这把锤子，看什么问题都像 Reward 设定的钉子。当 Agent 停滞不前时，我本能地将其归咎于“探索瘫痪”、“奖励错位”、“局部最优”等算法概念，甚至写了长篇大论去分析 Critic 网络是如何被 Shaping 奖励“骗”了的，却唯独没有去思考最朴素的问题：“它在物理上真的能开进去吗？”试图用算法（修改 Reward）去解决一个纯粹的物理碰撞问题，无异于缘木求鱼。

### 2.2 严重无视了项目已有的“血泪史”
在 `docs/diagnostic_reports/` 目录下，明明躺着 2 月份关于托盘碰撞体和插入深度的详尽报告。作为 AI 助手，我拥有检索整个代码库的能力，但在制定 0.2m 和 0.25m 的退火计划时，我竟然完全没有去查阅这些底层的物理限制记录，直接拍脑袋定下了阈值。

### 2.3 缺乏最基本的数学交叉验证
在设定阈值时，我没有花哪怕一秒钟的时间，把 `0.25m` 换算成实际的插入深度，并与已知的物理极限进行比对。

## 3. 造成的恶劣影响

1.  **算力与时间的巨大浪费**：从 Exp 5.5c 开始，到 5.5d、5.5e、5.6 以及刚刚启动的 5.7，这连续 5 个实验的基石都是错的。Agent 在一个不可能完成的物理牢笼里撞了几千个 Iteration 的墙。
2.  **污染了实验结论**：之前文档中记录的关于“0.25m 导致探索瘫痪”、“需要 5000 分大奖打破局部最优”的结论，其实都是建立在错误归因上的“伪科学”。（注：这些伪结论已在主文档中被修正）。

## 4. 痛定思痛：未来的铁律 (SOP 更新)

为了绝不再犯此类低级错误，确立以下铁律：

1.  **“遇事不决，先查物理”**：以后只要 Agent 出现断崖式的成功率归零，**第一动作绝对不再是改 Reward**，而是立刻运行 `verify_forklift_insert_lift.py --sanity-check` 进行物理极限排查。
2.  **“阈值必有算式支撑”**：以后设定的任何几何阈值（距离、角度），必须在计划中明确写出其对应的物理真实含义（例如：要求插入多少米），并与已知的物理极限进行比对。
3.  **“敬畏历史文档”**：在开启新的大阶段实验前，必须全局 grep 相关的历史诊断报告，避免重复踩坑。

## 5. 下一步行动修正

*   **废弃 Exp 5.7**：停止当前基于 0.25m 阈值和 5000 分大奖的无效实验。
*   **回归物理现实**：既然物理极限是 0.938m，我们必须将 `paper_rg_dist_thresh` 设定为一个大于 `1.2 - 0.938 = 0.262m` 的值。
*   **新计划**：将阈值设回 **0.28m**（要求插入 0.92m，物理可达），并结合 Exp 5.6 证明有效的“平衡 Shaping 权重”策略，完成最后阶段的训练。
