# 论文原生方法与 Master 基线双线复现计划 (去过度设计版 v5)

## 0. 铁律（历史教训）

1. **"遇事不决，先查物理"**：Agent 成功率归零时，第一动作是运行物理极限排查，而不是改 Reward。
2. **"阈值必有算式支撑"**：任何几何阈值必须换算成实际插入深度，并与物理极限（当前托盘最大插入 0.938m）比对。
3. **"敬畏但不迷信历史文档"**：开启新实验前，全局搜索相关的历史诊断报告，避免重复踩坑。但历史结论可能建立在当时有缺陷的环境或错误归因之上（如 Exp 3~6 的"推土机"部分源于相机盲区而非奖励），应以尊重但可批判的态度审视，结合当前最新环境状态重新判断。
4. **"传感器先于算法"**：在调整奖励函数之前，先确认传感器（相机角度、渲染质量、FOV）没有物理盲区。注意：Exp 3~6 的"推土机"行为主因仍是奖励函数和策略问题（Agent 从远处就歪着硬怼，此时托盘完全可见）。60° 相机盲区和货叉隐身是真实缺陷，但它们主要影响的是"插入末端的精准停车"，而非"全程的推土机策略"。修复传感器是必要的，但不能因此低估奖励函数重构的重要性。
5. **"RRL 必须冻结"**：分支 A 和分支 B 都必须严格 `freeze_backbone=True`。不冻结会破坏 ImageNet 预训练特征、拖慢训练速度（75s/iter vs 32s/iter），且偏离论文的 RRL 范式。

在创建 `dual_track_base` 之前，以下视觉环境修复必须已经在基座代码中生效：

| 修复项 | 旧值 | 新值 | 状态 |
|--------|------|------|------|

| 货叉渲染颜色 | 黑色（与地面融合） | **亮红色** `(1.0, 0.0, 0.0)` | ✅ 已修复（Exp 7 落地） |

| 动作空间 | 2D `action_space=2` | 2D（不变） | ✅ 已确认 |

| `rini` 逻辑 | `proj_vel < 0.05 且 dist_front > 0.3` | 不变（已对齐论文） | ✅ 已审计 |
| Backbone 冻结 | — | **必须 `freeze_backbone=True`** | ⚠️ Exp 7.1 误用了 false，需在双分支中严格锁定 |

**关键警告**：Exp 7.1 的启动脚本中使用了 `freeze_backbone=false`，这导致：
- 每 iteration 耗时 75 秒（冻结时约 32 秒），训练效率减半。
- ResNet34 的 2100 万预训练参数参与梯度更新，可能破坏 ImageNet 特征。
- Value Function Loss 在 15 个 iteration 内已飙升到 7000 万。
**双分支计划中必须显式锁定 `freeze_backbone=True`。**

## 2. 核心反思与分支定位

### 2.1 为什么势函数在 Master 成功，但在 Vision 下容易失败？（分支 A 的价值）

- **Master (15D Actor)**：输入是精确的物理坐标，容易建立精确映射。
- **Vision (Image Actor)**：输入是高维像素，映射困难且充满噪声。
- **Exp 7 的新启示**：60° 盲区和货叉隐身是真实的传感器缺陷，修复后视觉 Actor 的学习条件有所改善。但需注意：盲区只影响插入末端精度，Exp 3~6 的推土机主因仍是奖励函数——Agent 在托盘完全可见时就已经歪着硬推了。传感器修复是必要条件，不是充分条件。
- **分支 A 的定位**：作为**严格的控制变量基线**。在视觉环境修复后，如果势函数仍然失败，才能真正确认是"视觉+差分奖励"的组合不可行。

### 2.2 过去实验对"论文复现（分支 B）"的指导意义

- **Exp 5.9**：Agent 具备精确插入能力，但核弹权重(5000)导致 Value Function Loss 爆炸。
- **Exp 6.0/6.0b**：微小泛化即崩溃，证明"由窄到宽的课程学习"是死胡同。
- **叹息之墙**：0.15m/0.25m 的 rg 阈值物理不可达，浪费 5+ 个实验。
- **Exp 7 视觉修复**：60° 俯仰角导致插入末端盲区，货叉与地面融合导致"隐身"。这些传感器缺陷会加剧末端精度问题，但 Exp 3~6 的推土机行为主因仍是奖励函数（Agent 在托盘可见时就已歪着硬推）。传感器修复消除了一个干扰因素，使后续奖励函数实验的归因更纯净。

### 2.3 论文成功的核心解法

论文不是靠复杂补丁，而是：
- **把任务限制在可见、可驾驶、可控制的范围内**
- **双目 ResNet34 预训练 + 2D 极简动作 + 每 episode 动态生成的参考轨迹奖励**
- **$r_g$ 只是二值辅助奖励{0,1}，真正引导学习的是三项连续轨迹奖励**

我们的偏离与弥补：
- 单目相机（额外的深度歧义难度）→ 用参考轨迹的持续引导弥补
- Headless 渲染降级（低保真度）→ 红色货叉高对比度方案弥补

## 3. 双线分支具体行动点

### 3.1 分支 A：Master 势函数基线 (控制变量法)

- **目标**：在修复后的视觉环境中，验证势函数奖励是否可行。
- **核心配置**：
    - **动作空间**：2D（已确认）
    - **视觉基座**：ResNet34 + ImageNet 预训练 + **全程冻结**
    - **奖励函数**：Master 势函数（$\Phi_1, \Phi_2, \Phi_{ins}$ 差分奖励）
    - **初始分布**：与 Master 一致
    - **视觉环境**：75° 俯仰角 + 红色货叉（继承基座）

### 3.2 分支 B：论文原生方法复现 (去过度设计版)

- **目标**：用最纯粹的论文逻辑解决问题。
- **核心配置**：
    - **砍掉补丁**：删除 `w_align`、`w_band` 等软门控；废弃 $r_g$ 阈值退火；废弃空间衰减。
    - **$r_g$ 回归辅助角色**：权重降回与 $\alpha_1 \sim \alpha_3$ 同量级（如 1~5），阈值固定为 **0.28m**。
    - **参考轨迹奖励**：采用 **三次 Bézier + 直线（clothoid-lite）** 方案。
        - 每 episode 在 `_reset_idx` 中根据 spawn 位置动态生成。
        - 离散为 16~32 点 polyline 缓存，每步用 `torch.cdist` 做最近点投影。
    - **奖励形式**：优先使用 $\exp(-x/\sigma)$ 替代 $1/x$（最终由阶段 0 验证决定）。
    - **初始分布**：先保守后放宽，基于 FOV 可见性验证确定。
    - **视觉基座**：ResNet34 + ImageNet 预训练 + **全程冻结**
    - **视觉环境**：75° 俯仰角 + 红色货叉（继承基座）

## 4. 实验执行顺序

1. **创建公共基座与双分支**
    - 确认 Exp 7 的视觉修复（75° 俯仰角、红色货叉、ResNet34 支持）已在当前代码中。
    - 基于当前稳定代码拉取 `exp/vision_cnn/dual_track_base`。
    - 从基座分化出 `branch_a_master_potential` 和 `branch_b_paper_native`。

2. **分支 A 验证 (快速试错)**
    - 恢复 Master 奖励，运行基线训练，观察视觉环境修复后势函数的表现。

3. **分支 B 重构与验证 (主攻方向)**
    - **阶段 0：数学 + 可见性 + 渲染验证（半天）**
        1. 实现 Bézier + 直线轨迹生成算法。
        2. 随机生成 100 个初始位置，画出轨迹，检查几何正确性。
        3. 统计 FOV(90° 水平 + 75° 俯仰) 内托盘可见比例，确定安全初始分布。
        4. 抽取 Headless 渲染图像，确认红色货叉/白色托盘的对比度在各初始位置下均可区分。
        5. 验证 $1/r_{cd}$ 和 $\exp(-r_{cd}/\sigma)$ 的数值范围，决定最终奖励形式。
    - **阶段 1：代码清理与奖励重构**
        1. 砍掉门控、空间衰减、核弹奖励。
        2. 接入参考轨迹奖励（$r_{cd}$, $r_{c\psi}$）。
        3. 使用 debug_vis 在仿真中绘制轨迹进行视觉验证。
    - **阶段 2：保守分布下基线训练**
        1. 在安全初始分布下启动端到端 PPO 训练。
        2. 前 200~300 iterations（约 2 小时）做快速冒烟判断。

4. **深度复盘与超参调优**
    - 严格遵守"全链路反思"规则。
    - 重点调优 $\alpha_1, \alpha_2, \alpha_3$ 的相对权重。
    - 保守分布成功后逐步放宽。

## 5. 风险预判与缓解策略

| 风险 | 严重性 | 缓解策略 |
|------|--------|----------|
| 相机 FOV 盲区（近场托盘消失） | **已修复** | 75° 俯仰角（Exp 7 已验证） |
| 货叉与地面融合（Headless 渲染降级） | **已修复** | 红色货叉高对比度方案（Exp 7 已验证） |
| Backbone 未冻结导致训练崩溃 | **致命** | run_branch_*.sh 中显式锁定 `freeze_backbone=true` |
| $1/x$ 数值爆炸导致 Critic 崩溃 | **致命** | 阶段 0 验证 + 优先使用 $\exp(-x/\sigma)$ |
| rg 阈值物理不可达 | **致命** | 固定为 0.28m（已有物理极限算式支撑） |
| 远距离托盘在画面中太小 | 严重 | 保守初始分布（≤2.5m） + 阶段 0 渲染验证 |
| "轨迹抱死"（贴轨迹不前进） | 严重 | 监控 $r_d$ 改善速度，必要时提高 $\alpha_1$ |
| "0.3m 停车"（近场停滞） | 严重 | 监控 $r_g$ 触发率，必要时微调 $\alpha_4$ |
| $\alpha$ 超参数维度爆炸 | 中等 | 先固定惩罚项，只调 $\alpha_1:\alpha_2:\alpha_3$ 比例 |
| 单目深度歧义 | 中等 | 参考轨迹引导弥补；若不足，后备引入 Depth Camera |
