# 实验 5 系列：有界 Shaping 与消除探索恐惧

## 1. 背景与动机 (从实验 4.1 的失败中吸取教训)
在实验 4.1（近场课程验证）中，我们修复了 `dist_center` 的几何计算 Bug，并把出生点拉近到 0.3m ~ 0.6m。我们期望在这个极近的距离下，Agent 能够顺理成章地插进去。
然而，**实验 4.1 依然失败了**。Agent 变成了"近场木头人"，停在距离托盘 1.1m 的地方不敢动。

**深刻的教训：**
1. **$1/x$ 奖励形式是万恶之源**：即使在近场，只要 Agent 姿态稍微对准，$1/rcd$ 和 $1/rc\psi$ 就会提供巨大的奖励。此时，如果它往前走，哪怕稍微偏离一点点轨迹，分母变大，奖励就会断崖式下跌。这形成了一个**"局部最优黑洞"**。
2. **论文的成功具有局限性**：原论文能在 $1/x$ 下成功，可能是因为其操作空间极小（1.8m），且可能有未公开的 Reward Clipping 机制。在我们的环境中，直接使用 $1/x$ 会导致数值极度不稳定和严重的 Reward Hacking。

## 2. 实验 5：稀疏终局奖励 + 轻量有界 Shaping
为了彻底解决"刷分黑洞"，我们决定重构正向奖励函数。

### 2.1 核心假设
将所有 Shaping 奖励（距离、轨迹）从无界的 $1/x$ 替换为有界的指数函数 $exp(-x/\sigma)$。这样任何一项 Shaping 的最高分都被死死卡在 1.0。同时，赋予终局奖励（`rg`）极高的权重。这样，**"完成任务"在数学上将成为唯一的全局最优解**。

### 2.2 修改内容
1. **替换公式**：
   - `r_dist = exp(-dist_center / 1.0)`
   - `r_traj_d = exp(-(d_traj / 0.3)^2)`
   - `r_traj_yaw = exp(-(yaw_traj_err_rad / 0.3)^2)`
2. **调整权重**：
   - `alpha_4` (rg, 到达奖励) 从 10.0 暴增至 **200.0**。
   - `alpha_8` (rini, 停滞惩罚) 降回 5.0（因为 Shaping 有界了，不需要那么大的惩罚来逼迫）。
3. **保持近场课程**：出生距离依然在 0.3m ~ 0.6m。

### 2.3 实验 5 运行现象："探索瘫痪"
运行实验 5 后，发现 Agent **依然停在门外（1.1m）不敢进去**。
*   **原因分析**：虽然我们消除了 Shaping 的刷分漏洞，并挂上了 200 分的胡萝卜，但我们忽略了面前的"大棒"——**推盘惩罚 (`alpha_5 = 100.0`) 太高了**。
*   在随机探索初期，Agent 只要一靠近托盘，极大概率会发生碰撞，瞬间被扣 100 分。而它**从来没有**偶然触发过那 200 分的大奖。
*   因此，Critic 网络学到的经验是："前方是雷区，停在原地虽然拿不到大奖，但至少不会被炸死"。这是一种典型的**"探索瘫痪 (Exploration Paralysis)"**或"习得性无助"。

## 3. 实验 5.1：消除探索恐惧 (当前正在进行)

### 3.1 核心假设
在课程学习的初期（破冰期），必须**大幅降低探索成本**，并**强行把 Agent "喂"到嘴边**，让它体验到一次 200 分的快感。等它脑子里刻下了"插入=巨大奖励"的潜意识后，再在后续课程中慢慢恢复难度和惩罚。

### 3.2 实施计划
1. **极大降低推盘惩罚（消除恐惧）**：
   - 将 `alpha_5` (rp) 从 100.0 暴降到 **1.0**。允许它在初期"粗鲁地"推盘插入。
2. **进一步收窄初始随机范围（消除无知）**：
   - 将 Stage A 的初始偏角从 `±10°` 降到 **`±2°`**。
   - 将横向偏差从 `±0.15m` 降到 **`±0.05m`**。
   - 目的：让它出生就正对托盘，只要踩一脚油门就能触发 `rg`。

### 3.3 预期目标
这不是为了训练出一个能用的最终模型，而是为了**打破认知盲区**。只要 `push_free_success_rate` 能够快速破零并上升，就说明"破冰"成功，随后我们将无缝进入 Stage B（扩大姿态难度）、Stage C（扩大距离难度）和 Stage D（恢复推盘惩罚）。

## 4. 实验 5.2 - 5.5d：打通最后半米的"课程退火"之路

在实验 5.1 中，虽然我们消除了探索恐惧，但 Agent 依然停在门外。随后的连贯实验揭示了强化学习在极近距离下的复杂博弈。

### 4.1 实验 5.2：修复"原地鬼畜"与轨迹奖励喧宾夺主
*   **发现**：Agent 学会了以 0.06m/s 的速度原地摇晃，既逃避了停滞惩罚，又吃满了轨迹对齐奖励。
*   **修复**：将停滞惩罚改为"向托盘的投影速度"，并移除近场轨迹奖励，改用绝对姿态奖励。
*   **结果**：Agent 被迫前进，但变成了"推土机"（顶着托盘走，不插进去）。

### 4.2 实验 5.3 & 5.4：放宽 rg 阈值，打破认知障碍 (重大突破)
*   **发现**：Agent 变成推土机是因为 `rg` 触发条件太苛刻（误差 < 0.1m），在随机探索下物理不可达，导致 Critic 永远不知道"插入"有大奖。
*   **行动 (Exp 5.4)**：将 `paper_rg_dist_thresh` 从 0.1 放宽到 **0.4m**（插进一半就算赢）。
*   **结果**：**巨大成功！** Agent 偶然触发大奖后瞬间开窍，`rg` 触发率飙升至 60% 以上，回合长度大幅缩短。

### 4.3 实验 5.5 vs 5.5b：步子迈大了容易扯着蛋 (课程退火的艺术)
*   **行动 (Exp 5.5)**：在 0.4m 阈值稳定后，我们试图直接将阈值收紧到 **0.2m**，并加载 Checkpoint 继续训练。
*   **结果 (Exp 5.5 失败)**：`rg` 瞬间跌回 0。Agent 发现无法轻易拿到大奖，于是退化回了"推土机"状态（托盘位移飙升至 0.95m，偏航角恶化至 15°）。
*   **反思**：从 0.4m（插 2/3）到 0.2m（几乎插到底），对姿态精度的要求呈指数级上升。直接跳跃导致了"灾难性遗忘"。
*   **修正 (Exp 5.5b 平滑退火)**：
    1. 将阈值退回到 **0.3m**。
    2. 将推盘惩罚 `alpha_5` 从 1.0 稍微提高到 **3.0**，抑制推土机行为。
    3. 重新加载 0.4m 时的 Checkpoint。
*   **最终结果 (Exp 5.5b 成功)**：Agent 完美适应了 0.3m 的新难度。`rg` 成功率稳定在 50%~60%，且**偏航角误差显著下降**（从 15° 降至 8°，后期降至 5°），托盘位移也被压制在 0.3m 左右。

### 4.4 实验 5.5c：0.1m 步长依然太大，0.3m 直跳 0.2m 再次失败
*   **行动 (Exp 5.5c)**：在 5.5b (0.3m) 稳定后，将 `paper_rg_dist_thresh` 从 0.3m 收紧到 **0.2m**，加载 5.5b 的 Checkpoint 继续训练。`alpha_5` 保持 3.0 不变。
*   **结果 (Exp 5.5c 失败)**：训练了 **1278 个 Iteration（约 7.5 小时）**，`rg` 始终为 **0**，一次都没有触发过。
*   **但 Agent 精度达到了惊人的水平**：
    - 偏航角误差从 5.7° 降至 **1.6° ~ 2.5°**
    - 横向误差从 0.12m 降至 **0.05m**
    - 托盘位移从 0.33m 降至 **0.02 ~ 0.05m**（几乎零推盘！）
    - 对准率从 60% 升至 **86%**
    - `dist_front` 仅 **0.007m**（叉尖几乎贴住托盘前沿）
*   **关键发现**：Agent 学会了以"手术刀"级别的精度完美对准托盘口，但**死活不敢往里插**。因为它从未在 0.2m 阈值下触发过 `rg` 大奖，Critic 网络不知道"插进去"有 200 分的价值。它理性地选择了"完美站在门口不进去"这个局部最优。
*   **根本原因**：虽然 0.3m -> 0.2m 的跨度和 0.4m -> 0.3m 一样都是 0.1m，但**越接近完全插入，同样的 0.1m 对姿态精度的要求呈指数级上升**。0.1m 的步长在后半段不够平滑。

### 4.5 实验 5.5d：加入 0.25m 中间台阶 (失败)
*   **行动**：将 `paper_rg_dist_thresh` 设为 **0.25m**，加载 Exp 5.5c 的 Checkpoint。`alpha_5` 保持 3.0 不变。
*   **结果**：运行 1300+ Iteration，`rg` 依然为 0。Agent 保持了 2.8° 偏航和 0.009m 的贴脸距离，但依然不敢插入。

### 4.6 实验 5.5e：降低推盘惩罚试图破冰 (失败)
*   **行动**：在 0.25m 阈值下，将 `alpha_5` 从 3.0 降至 1.0，鼓励探索。
*   **结果**：Agent 尝试了推盘（位移升至 0.17m），但由于 0.25m 阈值较深，推盘过程中托盘滑动导致无法有效缩小相对距离，最终再次放弃探索，退回"贴在门口"的局部最优。

### 4.7 实验 5.5f：终极微步退火 (0.3m -> 0.28m) (当前正在进行)
*   **行动**：
    1. **退回起点**：加载 5.5b 的 Checkpoint (在 0.3m 阈值下有 50% 成功率的稳定状态)。
    2. **微步收紧**：将 `paper_rg_dist_thresh` 设为 **0.28m**（只比 0.3m 紧 2 厘米）。
    3. **保持精度**：恢复 `alpha_5 = 3.0`，防止退化为推土机。
*   **初步结果 (破冰成功)**：在启动仅仅 3 个 Iteration 后，`rg` 就成功触发（18.75%），随后稳步上升至 **25.0%**。这证明了"微步退火"（0.02m 步长）能够完美避免探索瘫痪。


### 4.5.1 深度解析：0.4m、0.3m、0.28m 到底意味着什么？
在我们的几何定义中，`dist_center` 是**"当前叉臂中心"**到**"目标叉臂中心"**的距离。
*   **目标叉臂中心**：当叉车**完全插入**（叉根抵住托盘前沿）时，叉臂中心所在的位置（位于托盘内部 0.6m 处）。
*   **当前叉臂中心**：叉车当前状态下，叉臂的几何中心。

因此，`paper_rg_dist_thresh` (触发大奖的距离阈值) 的物理含义是：**允许叉车距离"完美插入"差多少米**。
*   **阈值 0.4m**：允许差 0.4m。意味着只要叉车插入了 `1.2m(全长) - 0.4m = 0.8m`（插进约 2/3），就算成功。
*   **阈值 0.3m**：允许差 0.3m。意味着需要插入 `1.2m - 0.3m = 0.9m`（插进 3/4），算成功。
*   **阈值 0.28m**：允许差 0.28m。意味着需要插入 `0.92m`。
*   **阈值 0.1m (原论文设定)**：允许差 0.1m。意味着必须插入 `1.1m`（几乎完全插到底），才算成功。

**为什么前半段 (0.4 -> 0.3) 可以跨 0.1m，而后半段 (0.3 -> 0.2) 跨 0.1m 会失败？**
这本质上是一个**"容错空间与物理碰撞"的非线性关系**问题：
1. **前半段（相对宽松）**：在 0.4m 时，叉尖刚刚穿过托盘底部的横梁，此时托盘内部空间相对宽敞，只要大方向对，稍微有点偏航角（比如 10°），叉臂也不会卡死在托盘内部。从 0.4m 推进到 0.3m，虽然要求高了，但物理上依然有允许"微调"的余地。
2. **后半段（极度严苛）**：当要求插入深度超过 0.9m (阈值 < 0.3m) 时，叉臂已经深深没入托盘。此时，**哪怕只有 2° 的偏航角误差，在 1.2m 长的叉臂末端也会被放大成巨大的横向位移**，导致叉臂死死卡住托盘的内壁。
3. **探索与惩罚的死锁**：在 0.3m -> 0.2m 的跨度中，Agent 必须在**完全不碰到托盘内壁（否则会触发推盘惩罚 `rp`）**的前提下，把偏航角从 5° 压缩到 1° 以内，才能顺利滑入 0.2m 的深度。这在随机探索中是一个**概率极小的奇迹**。如果跨度太大（0.1m），它尝试了几千次都等不到这个奇迹，就会陷入"探索瘫痪"（贴在门口不进去）。
4. **微步退火的意义**：0.3m -> 0.28m（只要求多插 2cm），虽然容错空间也在缩小，但缩小的幅度在 Agent 当前的"能力射程"之内。它偶然一次姿态稍微好一点，就能滑进这 2cm 拿到大奖。拿到大奖后，Critic 就会强化这个"更直的姿态"，从而为下一步（比如 0.26m）打下基础。这就是**"用切香肠的方式克服指数级难度"**。

### 4.8 核心经验沉淀：动态退火课程学习 (SOP) - 最终版
实验 5.5b ~ 5.5d 的经验进一步完善了我们的**标准操作流程 (SOP)**：
1. **宽容破冰**：先用极宽的阈值（如 0.4m）让 Agent 尝到甜头，激活 Critic 网络。
2. **平滑退火**：绝不能一次性收紧到底。前半段（0.4 -> 0.3）可以用 0.1m 步长，**后半段（0.3 -> 0.2 -> 0.1）必须用 0.05m 步长**（因为精度要求呈指数级上升）。
3. **继承记忆**：每次收紧阈值，必须加载上一阶段稳定时的 Checkpoint，让"见过大奖"的 Critic 引导 Actor 提高精度。
4. **动态惩罚**：在收紧阈值的同时，可适当微调惩罚（如缓慢增加推盘惩罚），逼迫其放弃粗放动作，走向精细控制。
5. **失败的 Checkpoint 也有价值**：即使 `rg` 为 0，Agent 可能在失败的训练中学到了极高的精度（如 5.5c 的 2.5° 偏航角），这些精度会在下一阶段（更宽松的阈值）派上用场。

### 4.9 实验 5.5f 结果与反思：连坐机制与奖励错位
*   **结果**：微步退火（0.28m）成功触发了 `rg`，但成功率卡在 25% 左右，无法继续提升。
*   **深度分析（瓶颈所在）**：
    1.  **“连坐机制”（`rg` 的姿态门槛过严）**：`rg` 要求 `dist_center < THRESH & tip_y_err < 0.15 & yaw_err_deg < 10.0`。在深插时，物理碰撞会导致姿态不可避免地发生轻微变形（比如偏航角被挤到 11°）。此时距离虽然达标，但因为姿态超标，导致 `rg` 无法触发。这让 Agent 感到困惑。
    2.  **“奖励错位”（Shaping 权重失衡）**：距离奖励 `alpha_1 = 5.0`，而姿态对齐奖励 `alpha_2 = 1.0`, `alpha_3 = 1.0`。这导致 Agent 认为“往前冲”比“摆正姿态”重要 5 倍，从而倾向于“硬挤”而不是主动调整姿态。

### 4.10 实验 5.6：系统性 Reward 结构修正 (已完成)
*   **行动**：停止微调推盘惩罚，直接从底层逻辑上解决问题：
    1.  **放宽 `rg` 姿态门槛**：在 `env.py` 中，将 `tip_y_err < 0.15` 改为 `< 0.20`，将 `yaw_err_deg < 10.0` 改为 `< 15.0`。允许深插时有轻微的姿态变形。
    2.  **重塑 Shaping 权重**：在 `env_cfg.py` 中，将 `alpha_2` 和 `alpha_3` 从 1.0 提升到 5.0，与距离权重 `alpha_1` 齐平。用 Shaping 奖励逼迫它主动摆正车身。
    3.  **保持推盘惩罚**：`alpha_5 = 3.0` 不变。
    4.  **直接挑战 0.25m**：加载 5.5b 的 0.3m checkpoint，将 `paper_rg_dist_thresh` 设为 0.25m。
*   **结果**：经过 800 多个 Iteration，`rg` 依然为 0。但**底层姿态指标达到了前所未有的完美状态**：偏航角误差降至 5.5°，横向误差降至 0.1m，对齐率高达 75%。推盘位移也降至极低的 0.18m。

### 4.11 深度反思：撞上物理引擎的“叹息之墙” (重大纠偏)
**警告：之前关于“局部最优”和“探索瘫痪”的分析是建立在错误归因上的伪结论！**

通过重新运行 `verify_forklift_insert_lift.py --sanity-check` 物理极限排查脚本，并查阅 2 月份的 `pallet_dragging_issue` 历史报告，我们发现了一个极其荒诞的真相：**0.25m 的阈值在物理上是绝对不可达的！**

*   **物理真相**：当前放大了 1.8 倍的托盘，内部存在阻挡柱，加上物理引擎的凸分解膨胀，导致**货叉最多只能插入 0.938 米**就会被死死卡住。
*   **数学矛盾**：`dist_center < 0.25m` 意味着要求插入深度达到 `1.2 - 0.25 = 0.95m`。
*   **结论**：`0.95m > 0.938m`。Agent 根本不是不想插，而是它已经把货叉顶到了物理引擎允许的绝对极限，却依然够不到我们设定的那条虚幻的及格线。

这是典型的“算法傲慢”——试图用修改 Reward 的方式去解决一个纯粹的物理碰撞问题。详细的自我检讨和 SOP 更新请见 `critical_reflection_physics_wall_20260314.md`。
