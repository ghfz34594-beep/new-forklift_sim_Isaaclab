# Exp 7.1 对照实验：冻结 vs 解冻 ResNet34 (2026-03-17)

## 1. 实验背景

在完成相机盲区修复（75° FOV）、渲染材质修复（红色货叉）和成功判定逻辑修复（`insert_fraction = 0.40`）后，我们启动了 Exp 7.1 的训练。

初始方案采用**解冻（Finetune）ResNet34**，即允许 RL 梯度回传更新整个卷积网络。训练 400 代后发现模型陷入了"完美对齐但不敢插入"的局部最优解。

随后，参考论文《Visual-Based Forklift Learning System Enabling Zero-Shot Sim2Real Without Real-World Data》中采用的 RRL（Resnet as Representation for RL）方法——**冻结 ImageNet 预训练的 ResNet，仅训练后端 MLP**——我们启动了对照实验。

## 4. 分析与结论

### 4.1 为什么解冻模型"不敢插入"？
解冻 ResNet34 时，RL 的嘈杂梯度会持续微调卷积层的特征表示。这导致：
1. **特征漂移 (Feature Drift)**：在微距场景下（货叉贴近托盘口），视觉特征一直在变化，模型对"再往前一点会不会撞"缺乏稳定的判断依据。
2. **过度保守**：面对不稳定的视觉输入，模型选择了最安全的策略——停在安全距离外，稳稳拿到对齐奖励。
3. **推盘惩罚的威慑**：`alpha_5 = 3.0` 的推盘惩罚在特征不稳定时被放大，模型认为"冒险插入的期望收益为负"。

### 4.2 为什么冻结模型"暴力猛冲"？
冻结 ResNet34 时，ImageNet 的预训练特征在我们的简易渲染（红叉、白盘、灰地）下：
1. **粗粒度特征有效**：能提供足够的方向感和大致的空间位置信息（前 1200 代对齐率 90%+）。
2. **细粒度特征不足**：缺乏针对当前任务优化的精细空间分辨率。当模型开始探索插入策略时，它无法精确控制微距下的速度和角度。
3. **策略崩溃**：MLP 发现"精准对齐后缓慢插入"太难了（因为特征不够精细），转而发现"大角度猛冲"反而能偶尔成功插入并拿到 `rg` 奖励，于是策略向"暴力"方向演化。

### 4.3 核心矛盾
两组实验揭示了一个深层矛盾：

> **解冻 = 精准对齐 + 不敢插入**
> **冻结 = 敢插入 + 对齐崩溃**

理想的策略应该是两者的结合：**精准对齐 + 果断插入**。

## 5. 下一步方向

基于以上分析，有以下几个可能的改进方向：

### 方案 A：两阶段训练法
1. **阶段一（解冻，特征适应）**：使用解冻的 ResNet34 训练到对齐率 95%+ 稳定后，保存 Checkpoint。
2. **阶段二（冻结，策略微操）**：加载阶段一的 Checkpoint，冻结已经适应了当前渲染环境的 ResNet34，只训练 MLP 来学习插入微操。

### 方案 B：调整奖励平衡
- 提高 `alpha_4`（rg 奖励）到 200-500，增加插入的诱惑力。
- 降低 `alpha_5`（推盘惩罚）到 1.0-1.5，降低碰撞的恐惧感。
- 目标：让解冻模型在保持对齐的同时，认为"插入的收益 > 推盘的风险"。

### 方案 C：部分冻结（Partial Freeze）
- 冻结 ResNet34 的前 3 个 Stage（低级特征：边缘、纹理），只解冻最后 1 个 Stage（高级特征：空间关系）。
- 兼顾特征稳定性和任务适应性。
